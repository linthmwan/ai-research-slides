LingBot-Map: Geometric Context Transformer for Streaming 3D Reconstruction

Lin-Zhuo Chen*, Jian Gao*, Yihang Chen, Ka Leong Cheng, Yipengjing Sun, Liangxiao Hu, Nan Xue, Xing Zhu, Yujun Shen, Yao Yao, Yinghao Xu
arXiv:2604.14141v2GitHub Trending 2026-07-19Streaming 3D reconstruction

A feed-forward 3D foundation model that keeps SLAM-like geometric state inside learned attention: anchors, local dense window, and compact trajectory memory.

Sources: arXiv cover p.1; GitHub README / trending_discovery.md
TL;DR

The paper turns “streaming 3D memory” into an attention-design problem.

Instead of retaining every historical image token, GCA keeps full tokens only where they matter and compresses the rest into geometry tokens.

Abstract p.1; §1 p.2; §3.2 p.5-6
Problem

Streaming 3D reconstruction needs selective context, not just bigger context

  • Offline 3D foundation models process all views globally, so they are strong but not causal.
  • Recurrent streaming states forget geometry; causal caches keep redundant history and grow with sequence length.
  • Hybrid learned-SLAM systems add keyframes and pose graphs, but rely on hand-crafted selection plus iterative optimization.
  • Target: preserve scale grounding, local dense pose cues, and long-range drift correction with bounded per-frame cost.

Contributions

The contribution is a structured learned state, plus the training/runtime recipe to make it run

  1. GCA: anchor context + pose-reference window + trajectory memory in one structured attention mask.
  2. GCT: DINOv2 ViT features with alternating frame attention and GCA layers; camera token predicts pose, image tokens predict depth.
  3. Training: offline base model -> streaming GCA model, progressive views 24 to 320, context parallelism, relative pose loss.
  4. Runtime: FlashInfer paged KV cache, direct and VO modes, about 20 FPS at 518 x 378.
  5. Evidence: Oxford Spires, ETH3D, 7-Scenes, Tanks & Temples, NRGBD.
§1 p.2; §3.3 p.6-7; §4 p.8-11; §6 p.12-18
Method · GCA

GCA decomposes streaming memory into three roles

1. Anchor context

  • First n frames keep full attention and anchor token.
  • Fixes coordinate system and absolute scale before streaming begins.
  • Later frames attend to these fixed references.

2. Local window + 3. trajectory memory

  • Last k frames keep dense image tokens for local registration.
  • Evicted frames keep only camera / anchor / register tokens: 6 context tokens per frame.
  • Video temporal position encodings impose ordering on the trajectory memory.
GCA context = anchor full tokens + local full tokens + 6 tokens per evicted frame

Training

The recipe matters: start offline, then progressively learn long streaming context

Stage 1: base model

  • DINOv2 ViT patch size 14; 24 alternating attention blocks.
  • Global attention over 2-24 views.
  • AdamW LR 2e-4, 160K iterations, about 21,500 GPU hours.

Stage 2: streaming model

  • Replace global attention with GCA; Q/K/V weights transfer directly.
  • 160K iterations, LR 5e-4; views increase 24 -> 320.
  • Window k sampled 16-64; context parallelism dimension 16; about 15,360 GPU hours.
L = lambda_depth L_depth + lambda_abs-pose L_abs-pose + lambda_rel-pose L_rel-pose

Mechanism crown

Full history is not the same as useful memory.

GCA’s surprising result: evicting dense old image tokens can improve accuracy, because distant redundant frames inject attention noise while compact trajectory tokens keep the geometric signal.

Selective memory is both an efficiency trick and a denoising prior.

Limitations

What the paper still does not solve

  • No explicit loop closure: revisiting regions could still benefit from classical SLAM-style detection and correction.
  • Fixed-token memory: six retained tokens per frame may lose fine-grained geometry in tens-of-thousands-frame regimes.
  • No test-time optimization: challenging scenes might improve with bundle-adjustment-like refinement.
  • VO boundary drift: arbitrarily long sequences require window alignment, which can compound errors.
§7 p.19; §4.4 p.10
Engineering interpretation

Why image / 3D engineers should care

  1. A learned streaming model can inherit SLAM’s state taxonomy without inheriting its handcrafted optimization loop.
  2. KV-cache layout matters: paged cache plus sparse/structured attention is part of the model, not mere deployment detail.
  3. Long-horizon evaluation should include degradation deltas, not only per-dataset average scores.
  4. The best context is not always the largest context; benchmark full-history baselines as a falsification test.
