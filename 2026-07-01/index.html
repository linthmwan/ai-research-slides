SANA-Video: Efficient Video Generation with Block Linear Diffusion Transformer

Junsong Chen, Yuyang Zhao, Jincheng Yu et al. · NVIDIA / HKU / MIT / THU / PKU / KAUST
arXiv 2509.24695NVIDIA technical report2025-10-14

把 video DiT 的 bottleneck 從 quadratic attention 轉成 linear attention + constant-memory block KV cache。

Source: arXiv:2509.24695 p.1

這篇的重點不是「更大的 video model」，而是讓長影片生成的 memory slope 變平。

Linear DiT 解短片高解析速度；Block Linear Attention 解 1 分鐘長影片的 global context 與固定記憶體。
TL;DR from Abstract / §1 p.1-2

問題：video generation 先被 token count 壓垮

  • Wan 14B 生成 5 秒 720p 影片需處理超過 75,000 tokens，H100 上約 32 分鐘。
  • 長影片（>10s）若 full-sequence processing，attention / KV cache 成本會隨長度成長。
  • MAGI-1、SkyReel-V2 等 autoregressive video 方法探索長影片，但仍受 vanilla attention / local window 限制。
  • 核心問題：能否同時做到高解析、高品質、低成本、可部署到 cloud 與 edge？
§1 Introduction p.2

貢獻地圖：三個效率槓桿互相咬合

  1. Linear Video DiT：用 ReLU linear attention 取代 quadratic self-attention，加入 3D RoPE 與 temporal Mix-FFN。
  2. Block Linear Attention：把 causal linear attention 的累加 state 變成 fixed-size global KV cache。
  3. DCAE-V + data filtering：用 F32T4C32 深壓縮 VAE 降 720p token 壓力，再靠 motion/aesthetic/caption filters 補品質。
  4. Deployment：選擇性 NVFP4 quantization，RTX 5090 上 720p 5s 從 71s 降至 29s。
§1 p.2; §3.4 p.7; §5 p.10

系統總覽：SANA image backbone → video generator

架構選擇很務實：盡量保留 SANA T2I backbone，只補 video 所需的 temporal / long-context 模組。

Linear Video DiT：把 O(N^2) 改成 O(N)，但不能犧牲 temporal locality

  • Base: SANA Linear DiT + small decoder-only text encoder。
  • RoPE 放在 ReLU kernel 之後：RoPE(ReLU(x))，避免 ReLU 先抹掉 positional info。
  • 新增 temporal 1D conv 到 Mix-FFN，用 shortcut + zero init 保護 pretrained weights。
attention cost: full O(N2) → linear O(N)

影片解析度 / 長度拉高時，N 的成長比模型小幅變大更致命。

§3 p.3; §3.2 p.4-5

RoPE 的坑：linear attention 不是把 softmax 公式照搬就好

  • RoPE 直接作用在 Q/K 會破壞 ReLU output 的 non-negative property，分母可能接近 0。
  • 解法：numerator 保留 RoPE(Q/K)，denominator 移除 query 或 key 的 RoPE，保持 positive / stable。
  • 這頁是機制上最重要的細節：linearization 需要重新設計 positional encoding 的數值路徑。
Eq.2 / Fig.3 / §3.2 p.4-5

公式直覺：保留累加 state，而不是保留所有歷史 token

Oi = phi(Qi) · (Σj<i Sj + Si) / phi(Qi) · (Σj<i phi(Kj)T + phi(Ki)T)
  • 其中 Sj = phi(Kj)T Vj。
  • 只要 cache ΣS 與 Σphi(K)T，下一個 block 就能使用全局歷史。
  • 記憶體不再隨 video length 成長；長影片 generation 的工程瓶頸因此改變。
Eq.3 / §3.3.1 p.5

Causal Mix-FFN 也要配合：attention causal 但 conv leakage 不能漏掉

  • 每個 block 尾端加 zero padding，避免 training 時 temporal conv 看到未來 block。
  • 快取上一個 block 的 last token，提供下一個 block 的 kernel-size-3 temporal conv。
  • 最終 cache = attention cumulative state / cumulative keys + conv last-frame cache。
Fig.4 / §3.3.1 p.6

LongSANA：用 autoregressive block training 對齊 training / inference

  • 從 5s SANA-Video model 繼續訓練 LongSANA。
  • Monotonically increasing SNR sampler：later blocks 使用較大的 timestep，縮小 sampling space 並改善 block consistency。
  • Improved self-forcing：用 self-generated conditions 訓練，降低 exposure bias。
  • 因為 block linear attention 可用 constant-memory global cache，self-generation 長度可延到 1 min，而非被 local-window attention 卡住。

DCAE-V：720p 效率不是只靠 attention，latent 壓縮也很關鍵

  • DCAE-V: spatial down-sampling F=32, temporal factor T=4, channels C=32。
  • 32 channels 對齊 T2I latent space，讓 image backbone adaptation 更快。
  • 相較 Wan2.2-VAE 的 192 latent dimension，DCAE-V 對 small diffusion model 的預測負擔較低。

Data pipeline：低訓練成本仍能學 motion 的另一半

  • Scene cut: PySceneDetect + FFmpeg 切成 single-scene short clips。
  • Motion: Unimatch optical flow + VMAF；只留 moderate and clear motion，並把 motion score 注入 prompt。
  • Aesthetic / saturation: DOVER score + OpenCV HSV saturation filter。
  • SFT: 約 5,000 human-preferred videos，覆蓋 motion categories 與 visual styles。

RTX 5090 部署：NVFP4 不是全模型粗暴量化

  • Selective quantization: self-attn QKV/O projections、cross-attn query/output、FFN 1x1 conv。
  • Keep higher precision: normalization、temporal conv、cross-attn KV。
  • 720p 5s: 71s → 29s，2.4x speedup；作者稱品質與 BF16 baseline indistinguishable。
Fig.7 / §5 p.10

Ablation：三個小改動撐住 linear attention 的品質

  • 3D RoPE：attention map 更 local，training loss 更低。
  • Temporal conv：補 linear attention 較 dense / less local 的問題，改善 motion continuity。
  • Monotonic SNR sampler：比 random timestep sampling 更一致。
  • Attention type profile：解析度越高，linear attention 的 latency advantage 越明顯。

真正 work 的原因：linear attention 的 statefulness，不是只有 linear complexity。

它讓長影片的歷史資訊被壓成固定大小的累加 state；這同時改變 memory、latency、以及 self-forcing 可以 rollout 的長度。

限制與讀法：這是一篇很有工程價值的 tech report，但別把 claim 讀過頭

  • Publication status: 目前是 arXiv / NVIDIA technical report，沒有公開 reviewer discussion。
  • VBench + qualitative examples 不能完全代表 human preference 或 long-horizon failure taxonomy。
  • Default inference steps 的 latency 比較有價值，但不是完整 speed-quality Pareto frontier。
  • LongSANA VBench 沒有贏 Self-Forcing；它的主 claim 是 fixed-memory global cache 與 real-time 長影片。
  • DCAE-V clean reconstruction 指標不一定最好；優勢在 perturbation robustness 與 small-model latent fit。

給圖像/影片生成工程的啟發

  1. 效率不是單點優化：attention、VAE compression、data filter、quantization 要一起設計。
  2. Linear attention 要補 locality：RoPE placement、denominator stability、temporal conv 都是不能省的細節。
  3. 長影片的核心是 state design：只做 chunking/local window 會便宜，但會犧牲 global memory。
  4. Edge deployment 需要 selective precision policy：temporal conv / norm / cross-attn KV 這些地方不適合一刀切。
如果要做自己的 video backbone，這篇最值得學的是「把模型結構設計成能被 serving 系統喜歡」。
