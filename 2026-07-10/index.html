TurboBoA: Faster and Exact Attention-aware Quantization without Backpropagation

ICLR 2026 conference paper · Samsung Research · arXiv:2602.04929

Junhan Kim, Yeo Jeong Park, Seungwoo Son, Chungman Lee, Ho-young Kim, Joonyoung Kim, Yongkweon Jeon
ICLR 20261-bit / low-bit PTQAnnounced 2026-02-04
Source: PDF cover p.1; official ICLR poster page
TL;DR

Make BOA parallel enough without giving up its attention-aware correction.

TurboBoA groups out-channels, compensates them with closed-form updates, then repairs accumulated input error and stale quantization grids.

Abstract p.1; §1 p.2
Problem

PTQ has a speed-accuracy split at very low bit-width

  • GPTQ is fast because it treats rows/layers independently, but INT2 accuracy degrades sharply.
  • BOA uses attention-aware Hessians and cross-channel error compensation, improving accuracy.
  • The cost: BOA processes out-channels sequentially; 128 out-channels can mean 128 sequential steps.
  • The paper asks: can we keep BOA-like compensation but reduce the sequential bottleneck?

Contributions

Three changes, one pressure point

  1. F1: joint multi-out-channel quantization. Quantize N rows at once and update remaining rows exactly.
  2. F2: pre-quantized-layer error correction. Add the input-deviation term caused by earlier quantized layers.
  3. F3: adaptive grid + CD refinement. Recompute scales after weights move, without backpropagation.
  4. Experiments: >3x speedup over BOA and better PPL / zero-shot accuracy on Llama-family low-bit PTQ.

Method F1

Joint quantization with exact compensation

  • Take N out-channels as a block B and quantize them simultaneously.
  • Do not discard BOA’s attention-aware Hessian; use the Kronecker-structured Hessian H = Hin ⊗ Hout.
  • Update the remaining rows using the Cholesky factor of Hout-1, so grouped errors are still compensated.
  • The authors empirically find reduced degrees of freedom are not harmful up to large N.
minimize ||G ΔW X||F2, subject to eiTΔW = Qi,: - Wi,: for i in B
Prop.3.1; §3.1 pp.3-4
Method F2

Later layers inherit distorted inputs

  • BOA compensates current-layer quantization error but not the distribution shift caused by already-quantized earlier layers.
  • TurboBoA stores the difference between current quantized input X and full-precision input XFP.
  • The compensation objective now includes both current weight error and propagated input error.
  • This is the low-bit regime’s hidden failure mode: a good local row update can still chase the wrong input.
GQX - GWXFP = GΔWX + GWΔX, where ΔX = X - XFP
§3.2 pp.4-5
Method F3

The quantization grid must move with the updated weights

  • BOA initializes a grid once, but compensation keeps moving weights afterward.
  • TurboBoA computes the grid immediately before each out-channel group is quantized.
  • After assigning integer weights Wint, it refines only the scales s by coordinate descent.
  • One CD iteration is used in main experiments because extra iterations provide marginal gains.
minimize over s: ||G(diag(s)Wint - W)X + GWΔX||F2
§3.3 pp.5-6; §4.1 p.7
Algorithm spine

TurboBoA in five operations

  1. Compute attention-aware Hessians Hin, Hout for Q/K/V projection weights.
  2. Compute input-deviation term R = α(X - XFP)XT.
  3. For rows i = 0, N, 2N, …: choose N out-channels B.
  4. Quantize B, then apply the closed-form update to the remaining out-channels.
  5. Refine scales by CD and output Q = diag(s)Wint.
No backpropagation loop is introduced; the paper stays in the PTQ / closed-form family.
Algorithm 1 p.5; Prop.3.1-3.3
Experimental setup

Evaluation is standard low-bit Llama PTQ

  • Models: Llama3.2-1B / 3B, Llama3-8B, Llama3.1-70B, Llama2-7B / 13B.
  • Calibration: 128 WikiText-2 sequences, length 2048.
  • Metrics: Wiki2 and C4 perplexity; average accuracy over eight zero-shot commonsense tasks.
  • Hardware: NVIDIA H100 80GB; the 70B model uses two GPUs.

Practical interpretation

Why N=16 is the conservative sweet spot

  • Increasing N lowers sequential steps, but also leaves fewer remaining rows to absorb grouped quantization error.
  • Empirically, degradation is negligible up to N=64, but speed gains flatten after N=16.
  • The authors therefore use N=16 for main experiments to preserve compensation flexibility.
  • For very large models, the absolute-time saving is already large at N=16.
§4.2 pp.7-8
What changed vs BOA

TurboBoA is a drop-in algorithmic upgrade, not a new inference graph

  • It modifies the quantization procedure, not the final model architecture.
  • It remains backpropagation-free and uses closed-form updates / CD scale refinement.
  • It can combine with transformation-based outlier suppression such as QuaRot, SpinQuant, and OSTQuant.
  • The paper frames it as a path toward low-bit deployment on constrained hardware.
Abstract p.1; §4.3 pp.9-10; §5 p.10
Limitations

Where I would be careful

  • The formal error bound as a function of group size N is left open by the authors.
  • F2 needs an extra forward pass to compute full-precision activation deviation; this is one-time, but not free.
  • Main-body evidence is Llama-family heavy; generality to very different architectures is less established.
  • The method inherits dependence on attention-aware Kronecker-structured Hessian quality.
§4.1 p.6; §4.2 pp.7-8; §5 p.10
Engineering takeaways

Three ideas worth stealing

  1. If an accurate algorithm is sequential, ask whether exact compensation can be moved to the remaining degrees of freedom.
  2. For quantization, local reconstruction is insufficient when earlier quantized layers distort the inputs.
  3. Scale/grid updates can be cheap if integer assignments are frozen and only continuous scales are refined.
The design taste is elegant: accelerate the bottleneck, then add just enough math to keep the original objective honest.

Three formulas to remember

  • Current-layer reconstruction: ||G ΔW X||F2.
  • Propagated input distortion: GQX - GWXFP = GΔWX + GWΔX.
  • Scale refinement: minimize ||G(diag(s)Wint - W)X + GWΔX||F2.
