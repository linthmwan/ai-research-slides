AnyBCQ: Hardware Efficient Flexible Binary-Coded Quantization for Multi-Precision LLMs

Gunho Park, Jeongin Bae, Beomseok Kwon, Byeongwook Kim, Se Jung Kwon, Dongsoo Lee - NAVER Cloud
ICLR 2026 conference paperarXiv:2510.10467v2Compression / Quantization

一份把 multi-precision LLM 從「可切 bit」推進到「硬體也真正吃得到 bit-plane」的量化工作。

Sources: arXiv PDF/HTML, OpenReview forum XPIEkFdEDi, GitHub naver-aics/anybcq
TL;DR

AnyBCQ 的本質不是又一個低 bit quantizer；它把 2/3/4-bit flexibility 做成 shared binary bit-planes + per-precision scales，讓 kernel 可以直接在 bit-plane 上算。

4.99GB vs 9.85GB2-bit MMLU 35.32 vs 24.66up to 4.07x GEMV speedup
Abstract p.1; Table 1 p.3; Table 2 p.8; Table 3 p.9
Problem

Multi-precision 的承諾：一個模型，動態 SLO

  • 現實限制：LLM deployment 同時被 memory footprint 與 latency 卡住，weight-only PTQ 實用但通常固定精度。
  • SLO 不是固定的：不同 request 可接受不同 accuracy-latency trade-off，因此需要單一模型在多個 bit-width 下運作。
  • 舊路線代價：每個精度各存一份模型太胖；Llama-3.1-8B 的 2/3/4-bit multi-model footprint = 9.85GB。
壓縮不是只看平均 accuracy；要看「同一份部署物」能否在 runtime 切換 precision。
§1 p.1-3; Table 1 p.3
Why prior multi-precision is awkward

Non-uniform quantization 會卡在 centroid lookup

Any-Precision LLM style

  • bit-planes 可用來存 index
  • 但 inference 要把 bit-planes 轉回 centroid index
  • bit-transpose + table lookup 讓低 bit 的 bandwidth 優勢被吃掉

AnyBCQ style

  • binary planes 本身就是可執行表示
  • B ∈ {-1,+1}，乘法近似 add/sub
  • 載入前 p 個 planes 即 p-bit inference
核心問題不是「能不能存 bit-plane」，而是「compute path 能不能不離開 bit-plane」。
§3.3 p.6-7; Fig.3 p.6
Contributions

三個貢獻共同鎖住 accuracy-memory-speed triangle

  1. Progressive precision expansion: 從 base precision 開始，逐步加入 residual bit-plane，舊 binary codes frozen。
  2. Shared binary + precision-specific scales: 2/3/4-bit 共用 binary payload，只為每個 precision 保留 scale set。
  3. Specialized CUDA kernel: 直接 bit-plane operations，支援 per-request precision selection，避開 centroid reconstruction。
  4. Empirical frontier: 2-bit accuracy 大幅改善，3/4-bit 保持競爭力，throughput 全面高於 AP baseline。
§1 p.3; §3.2 p.5-6; §3.3 p.6-7; §4 p.7-10

Progressive expansion：從 2-bit base 長到 4-bit target

Fig.1 p.2-3; §1 p.3
BCQ representation

把 multi-bit weight 寫成 binary operations 的序列

W_hat(p) = Σi=1..p αi,p Bi, Bi ∈ {-1,+1}
  • 每個 bit level 決定一個 binary assignment。
  • p-bit inference = 前 p 個 binary bit-plane computations。
  • scale 是 FP16；binary 是 INT1，因此非常適合 bit-parallel arithmetic。
BCQ 的「受限表示」換來的是更乾淨的 kernel mapping。
§3.1 p.4-5; Fig.2 p.5
Algorithm

Expansion 的關鍵：freeze old B, learn new residual plane

  1. Base precision: greedy 初始化 scales 與 binary codes。
  2. Refinement: 在 T cycles 中用 LS 更新 scales，再用 BS 重新分配 base binary codes。
  3. Higher precision: 新 scale 初始化為 0，新 bit-plane 由 residual sign 初始化。
  4. Shared constraint: 舊 binary codes frozen；只更新 scales 與新增 plane，避免多份 binary storage。
這是 accuracy 與 memory 的折衷點：越高 bit，shared-binary constraint 會稍微限制 peak accuracy。
Algorithm 1 p.5-6; §3.2 p.5-6; Table 6 p.14
Calibration

MRE/block reconstruction 是 2-bit 不崩的基礎

Optimization settings

  • Calibration: 512 sequences from C4.
  • Group-wise asymmetric BCQ, group size g = 128.
  • 10 epochs, learning rate 1e-4.
  • Refinement cycles T = 20.

What is optimized

  • Per-target precision scales are maintained separately.
  • Reconstruction error minimization updates scales for current precision.
  • Bit-planes are fixed during scale optimization.

My read

這篇最值得學的地方，是它沒有把「模型壓縮」和「kernel 實作」分開看。

AnyBCQ 用較受限的 BCQ representation，換到可直接執行的 bit-plane algebra。對部署而言，這個交易很乾淨：少一點高 bit peak accuracy，換多 precision、低 memory、低 overhead。

§3.3 p.6-7; §7 p.11
Limitations

漂亮，但不是沒有代價

  • Theory gap: progressive precision expansion 目前主要靠 empirical evidence，缺少嚴格保證。
  • High-bit ceiling: BCQ + shared-binary constraint 會降低高 bit-width 的表達能力；4-bit 有時輸給 non-uniform AP。
  • Hardware scope: A100/H100 證據強，但針對 iFPU / FIGLUT 類 accelerator 的更大收益仍是合理期待，不是本文實測。

Engineering takeaways

如果你在做 LLM serving / quantization，該記住這三點

  1. Representation is an interface contract. Quantizer 的 output 是否能被 kernel 直接吃，常比單點 accuracy 更重要。
  2. Multi-precision 的真成本在 shared payload. 如果 binary payload 不能共享，多 precision 只是多份模型的包裝。
  3. 2-bit 必須作為一等公民。 一旦 2-bit 不穩，mixed-precision decoding 的平均 bit-width 就很難真正壓低。
也就是說，量化演算法要從一開始就把 memory layout 和 kernel path 放進目標函數裡。

Appendix highlight

原文螢光筆：bit-plane-level computation

“By representing weights as binary bit-planes with corresponding scale factors, AnyBCQ enables bit-plane-level computation and maps naturally to accelerator-friendly, bit-parallel arithmetic.”

AnyBCQ 的關鍵是把權重表示成 binary bit-plane + scale，使 compute path 直接對齊硬體友善的 bit-level arithmetic。Abstract p.1

“AnyBCQ reduces the total memory footprint by 49% compared with the multi-model baseline on Llama-3.1-8B.”

同時支援 2/3/4-bit 不需要三份模型；共享 binary terms 讓 footprint 從 9.85GB 降到 4.99GB。Table 1 / §1 p.3

Appendix highlight

原文螢光筆：overhead 與 limitation

“bit transposition is the dominant overhead, accounting for roughly 35-58% of the latency...”

速度優勢的機制有硬證據：AP 的 bit-transpose 本身就是大頭 overhead；AnyBCQ 是把這步拿掉。Appendix A.2 / Table 7 p.14

“A limitation is that the inherent representational capacity of BCQ, together with the shared-binary constraint, can reduce peak accuracy at higher bit widths relative to non-uniform schemes.”

不是所有地方都贏；高 bit-width 下 BCQ 的表達能力與共享 binary constraint 會吃掉一點峰值 accuracy。§7 p.11

今日結論

AnyBCQ 是一篇很「部署導向」的量化論文：它把多精度、低記憶體、kernel 可執行性放在同一個設計裡，而不是各自最佳化。
