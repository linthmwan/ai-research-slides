ERNIE-Image：用資料挖掘與監督品質挑戰「模型尺寸 scaling law」

Jiaxiang Liu, Zhida Feng, Pengyu Zou, Zhenyu Qian, Tianrui Zhu 等 · Baidu ERNIE Team
arXiv 2605.25347v1Baidu · Technical ReportAnnounced 2026-05-25

8B 開源 single-stream DiT 文生圖模型，不靠堆參數、改靠資料管線與對齊品質，逼近 Nano Banana 2.0 / Seedream 等閉源 SOTA。

來源：arxiv.org/abs/2605.25347 · ERNIE-Image Technical Report
今日主題 · TL;DR

與其把 T2I 模型越堆越大，ERNIE-Image 證明：更深的資料挖掘 + 更高的監督品質，能讓 8B 模型在指令跟隨、文字渲染、美學生成上逼近頂級閉源系統。

GenEval 0.89（全表最高）· LongText-Bench 0.973（開源第一、反超 Nano Banana）· 人類偏好開源第一、整體第二 — 全部用 8B 參數達成，並開源權重 + 美學模型 + ERIA-1K benchmark。
問題背景 · §1 Introduction

閉源壟斷下，開源的兩條路都有缺陷

路線 A — 堆參數
  • 代表：Qwen-Image、HunyuanImage
  • 邊際報酬遞減、算力門檻高
  • 限制社群普及與可近用性
路線 B — 小而精
  • 代表：6B Z-Image
  • 社群認可、輕量易用
  • 但複雜指令跟隨與中文字渲染明顯受限
ERNIE-Image 的定位：open-source × powerful × easy-to-use 三者兼得，補齊開源最缺的 complex instruction following / text rendering / aesthetic generation。
核心貢獻 · 三大支柱

把標題拆成三個命題：資料、監督、美學

  1. Data-centric pre-training：bottom-up 資料建構（10K 細粒度分類 + caption + 美學評分 + 階層採樣），取代盲目擴參數
  2. Supervision-quality post-training：SFT（prompt 多樣化）→ Stabilized DPO（anchor loss 防 reward hacking）→ MT-DMD（多老師蒸餾防能力漂移）
  3. Aesthetics as infrastructure：自建工業級 ERNIE-Image-Aes 美學模型 + ERIA-1K benchmark，美學評分回頭驅動資料清洗

關鍵 insight：在訓練早期就注入「詳細圖像描述」，不只強化複雜指令跟隨，還大幅豐富 world knowledge → 因此不刻意合成領域資料，改用更精準的標註策略挖掘海量但低品質的圖庫。

方法 · 架構 · §1 p.2

8B Single-Stream DiT，刻意選小型 text encoder

  • Latent Diffusion Model，single-stream DiT，8B 參數
  • VAE：FLUX.2 VAE（強開源 latent space）
  • Text encoder：Ministral-3 (3B) — 刻意選小型，降記憶體 + 降使用門檻
衍生模型家族
  • ERNIE-Image-Turbo（8-NFE 快速生成）
  • Prompt Enhancer（3B 輕量）
  • ERNIE-Image-Aes（工業級美學模型）
實驗證實：即使 text encoder 只有 3B，仍能撐住長而複雜的指令輸入 — 能力來自資料與訓練，而非編碼器規模。
方法 · 預訓練 · §2.1 p.3

Bottom-up 資料管線：對海量原始圖庫逐步施加結構

  1. Fine-grained categorization：分類模型把每張圖歸入 10,000 個細粒度視覺類別 → 保住 long-tail 概念、避免主流類別淹沒分佈
  2. Caption enrichment：VLM 為全部圖生 caption，特別強調忠實辨識並描述圖中文字（slides / 圖表 / 海報 / UI / 文件）— 文字漏標會嚴重削弱 image-text alignment
  3. Aesthetic scoring：用 ERNIE-Image-Aes 給全部圖打美學分，作為大規模過濾/採樣的統一品質訊號（學習式品質估計 > 啟發式規則）
  4. Hierarchical sampling：兩階層採樣（見下頁）

核心對比：不從預定資料分層 resample（top-down），而是 bottom-up 從原始分佈逐步建構結構。

方法 · 預訓練採樣 · §2.1 p.3

兩階層採樣 + 三階段解析度課程

Hierarchical sampling
  • inter-category：採樣權重 = 語料規模 + 整體美學品質 → 高頻類別不獨大、高品質類別分到更多機率質量
  • intra-category：類別內按美學分採樣，高品質實例曝光更頻繁
漸進解析度課程
256×256  →  512×512  →  1024×1024
  • 全程訓練多種 aspect ratio（不限正方形）
  • 早期低解析優化效率 + 後期保留高解析靈活構圖
美學評分是貫穿全管線的「統一品質訊號」：既決定類別間機率質量，也決定類別內曝光頻率。
後訓練 A · SFT · §2.2.1 p.4

SFT 反向操作：top-down 策劃高一致性資料

領域優先策劃
  • 與預訓練 bottom-up 相反：先定優先領域再策劃資料
  • 優先領域：poster design、game screenshots、portrait / object photography、anime-style 等高需求類別
  • VLM 生 caption：內容、場景佈局、物件屬性、可見文字的結構化監督
Prompt 多樣化（關鍵）
  • 用更大的 VLM（K2.5）把 caption 改寫 + 降採樣成多樣 prompt
  • 同一張圖 → 短關鍵詞 / 自然語言 / 指令式 / 詳細構圖式等多形式
  • 縮小「合成 caption ↔ 真實使用者 prompting」落差
目的：提升對異質使用者輸入的 robustness — 真實使用者不會像訓練 caption 那樣下 prompt。
後訓練 B · Stabilized DPO · §2.2.2 p.5-6

在 DiT 上解掉 DPO 的 reward hacking

  • 在 Flow Matching 框架下，DPO 從 velocity-field 觀點重構：以預測 velocity vθ 的 L2 重建誤差定義 implicit reward
  • 核心問題：naive DPO 利用 L2 loss「無上界」特性，瘋狂放大 rejected 樣本誤差 → representation collapse
LDPO = −𝔼[ log σ( −β ( Diffpolicy − Diffref ) ) ]   （式2）
Ltotal = LDPO + λwin·𝔼[ℓpolicywin] + λlose·𝔼[ℓpolicylose],   ℓ = ‖ vθ(xt,h,t) − vt ‖₂²  （式3）

Anchor Losses 把模型基本生成能力「錨定」住防止崩塌。超參：β = 0.05、λwin = 0.35、λlose = 0.15。

後訓練 B · 機制觀察

當 base 模型與 reward 模型都夠強時，只需極少 DPO steps 就能對齊到位 — 而越少步、越能最小化 reward hacking 的暴露窗口。

啟發：DPO 的穩定性不只靠 loss 設計，也靠「強 base + 強 reward + 短訓練」三者共振。anchor loss 是兜底，不是主力。
後訓練 C · MT-DMD 少步蒸餾 · §2.2.3 p.6-7

多老師蒸餾：解掉單老師的 Capability Drift

  • 背景演進：DMD → DMD2（score model 更新 5 次才更新 generator 1 次）→ DMDR（RL + dynamic guidance + step-aware LoRA scaling）
  • 單老師痛點 Capability Drift：處理高度異質資料時，單一 teacher 即使有 dynamic LoRA + reward，也難在所有語意域給一致最優監督（清晰排版 vs 風格化美學互相拉扯）
  • 解法 MT-DMD：編排「專家老師委員會」ℰ = {E₁,…,EK}，各自專精不同域：Text-Rendering Expert（排版）/ Digital Art Expert（風格化）/ Spatial Layout Expert（宏觀構圖）
把多步 diffusion 蒸餾成 8-NFE 的 ERNIE-Image-Turbo，同時保住跨域能力，不被單一 oracle 的瓶頸限制。
後訓練 C · 機制皇冠頁 · §2.2.3 p.7

Dynamic Routing Manifold：樣本域 × timestep 的多工

  • 把「樣本域 + diffusion timestep」當成統一高維 multiplexing 空間 W，動態路由到對的專家
  • 非對稱梯度拓撲：同一訓練實例內，DM 懲罰查 Digital Art Expert 顧全域風格一致，CA 約束同時查 Text-Rendering Expert 顧局部拼字/結構
  • 沿 diffusion 軌跡無縫 expert handoff：高噪聲階段用 Spatial Layout 專家立宏觀構圖；低噪聲階段交棒給高頻渲染專家（寫實光照 / 材質紋理）
路由不是「一張圖配一個老師」，而是「在去噪軌跡的不同階段、不同子目標上分別調用最合適的專家」— 消除單 oracle 的能力天花板。
美學基礎設施 · ERNIE-Image-Aes · §3 p.8-10

自建工業級美學模型，因現成預測器有系統性偏誤

現成模型的偏誤
  • LAION-Aes：對 AI 生成 / 動漫 / 隨手拍給過高分
  • ArtiMuse / UniPercept：偏好黑白照、對日常活動快照（公園素人合照）誤判高分
標註方法
  • 採 pairwise comparison（兩兩比較）而非 absolute Likert → 規避 score drift
  • 用 Swiss-system tournament 做局部二元判斷，更穩定可複現
  • 標註者須先過 aesthetic calibration 測驗；每人獨立完成整個 tournament，標籤在 dataset 層級聚合

模型從 ArtiMuse（8B VLM） 初始化，在內部標註資料微調；訓練資料沿「美學品質先驗 × 內容類別」兩軸保證多樣性。

個人解讀 · 對圖像 / 多模態工程師的啟發

五個可複製到日常工作的 takeaway

  1. 資料管線 > 參數量：bottom-up 細分類（10K 類）+ 學習式美學評分 + 階層採樣，是可複製的資料品質工程，比盲目擴模型更划算
  2. caption 要忠實標出圖中文字：文字漏標直接削弱 alignment — 做 text rendering 必備的標註紀律
  3. DPO 上 DiT 要防 reward hacking：anchor loss 錨定基本能力（β=0.05, λwin=0.35, λlose=0.15 可直接參考）
  4. 少步蒸餾用多老師：按「域 + timestep」routing 勝單老師，避免 capability drift
  5. 美學評分用 pairwise + calibration 篩選標註者，比 Likert 絕對分穩定 → 自建品質訊號可回頭驅動資料清洗

資料與監督的勝利，不是參數的勝利

ERNIE-Image 用 8B 開源模型，靠 bottom-up 資料挖掘、stabilized DPO、MT-DMD 多老師蒸餾與自建美學基礎設施，逼近 Nano Banana 2.0 / Seedream — 並開源權重、Turbo、Prompt Enhancer、美學模型與 ERIA-1K benchmark。

附錄 · 螢光筆精華 (1/3) · 核心論點

挑戰 model-size scaling law

We challenge the scaling law of model size and show that further mining pre-training data can push beyond the limits of conventional understanding. By introducing more complex and detailed instructions at an early stage, the model naturally develops precise instruction following and strong text rendering.

我們挑戰模型尺寸的 scaling law，證明更深入挖掘預訓練資料能突破傳統認知的極限。透過在訓練早期就引入更複雜、更詳細的指令，模型自然湧現出精準的指令跟隨與強大的文字渲染能力。§5 Conclusion p.22

附錄 · 螢光筆精華 (2/3) · 資料管線

Bottom-up 資料建構 + 10,000 細粒度類別

We adopt a bottom-up data construction and sampling strategy. Unlike top-down resampling pipelines that operate on predefined data strata, our pipeline starts from a massive raw image corpus. We first train an image categorization model to assign each image into one of 10,000 fine-grained visual categories … serving as the basis for data balancing in subsequent stages.

我們採用由下而上（bottom-up）的資料建構與採樣策略。不同於在預定資料分層上操作的 top-down resampling 管線，我們的管線從海量原始圖庫出發。首先訓練一個分類模型，把每張圖歸入 10,000 個細粒度視覺類別之一……作為後續階段資料平衡的基礎。§2.1 Pre-training p.3

附錄 · 螢光筆精華 (3/3) · DPO 穩定化

Reward hacking 與 Anchor Losses

A significant challenge in applying naive DPO to DiT models is reward hacking, where the model exploits the unbounded nature of L2 loss by excessively increasing the error scale of rejected samples. This often leads to representation collapse. To mitigate this, we introduce Anchor Losses as a regularization mechanism to ground the model's fundamental generative capability.

在 DiT 模型上套用 naive DPO 的一大挑戰是 reward hacking：模型利用 L2 loss 無上界的特性，過度放大被拒樣本的誤差規模，常導致表徵崩塌（representation collapse）。為緩解此問題，我們引入 Anchor Losses 作為正則化機制，把模型的基本生成能力「錨定」住。§2.2.2 Stabilized DPO p.5-6
