MiniMax Sparse Attention

Xunhao Lai, Weiqi Xu, Yufeng Yang, Qiaorui Chen, et al. · MiniMax · Peking University · NVIDIA · Zhejiang University · HUST
Company Tech ReportarXiv:2606.13392Announced 2026-06-11
在 GQA 上做 blockwise 稀疏注意力：每個 group 各自選 Top-k 個 KV block，1M context 下 attention FLOPs 降 28.4×，能力與 full attention 打平。
TL;DR

一個極輕量 Index Branch 對 KV block 打分、每個 GQA group 各自選 Top-k；Main Branch 只對選中 block 做精確 softmax。109B MoE 上與 full-attn 打平，1M context FLOPs 降 28.4×、prefill 14.2×、decode 7.6×。

設計哲學 = Occam's razor：大量 ablation 後只留必要組件，可 from-scratch 訓練、也可從既有 GQA checkpoint near-lossless 轉換。
問題背景

Ultra-long context 撞上 O(N²) 的牆

  • Agentic workflow、repo-scale code、persistent memory 要 attend 數十萬~百萬 token，softmax attention 的 O(N²) 在部署規模不可行。
  • Hybrid / linear / SSM（Mamba）：換掉部分 exact-softmax 能力，長程精確檢索受限。
  • Fixed-pattern（local window / global token / sink）：content-agnostic，選擇與內容無關。
  • Inference-time sparsification（H2O / SnapKV / Quest / MInference / InfLLM）：繼承 full-attn 訓練成本，至少一個 inference 階段仍接近 dense。

定位

Natively-trained sparse 的鄰居，與 MSA 的兩軸差異

  • NSA：三條並行分支（compressed / selected / sliding window），較重。
  • MoBA：GQA 上用很大的 KV block、block-averaged key 打分，indexer 只靠 LM gradient 訓。
  • DSA：建在 MLA(MQA mode) 上，token-level 選擇、所有 head 共用單一 Top-k。
  • MSA：per-GQA-group Top-k 共享 + block-level 選擇兩軸一起取 — 兼顧 multi-group block-granular retrieval 與 contiguous KV reads。
§6 Related Works p.13
核心貢獻

MSA：minimal · scalable · accelerated

  1. 提出 MSA — 建在 GQA 上的 blockwise sparse attention，支援 from-scratch 訓練與從 GQA checkpoint near-lossless 轉換。
  2. Co-design 訓練 + 推論 kernel，把理論 FLOP 節省化為真實 wall-clock 加速（exp-free TopK、KV-outer sparse attention）。
  3. 在 109B MoE + native multimodal 上做大量 ablation，跨 scale 與 modality 解剖 MSA 行為，得出最小必要設計。

方法 · 命題 1 / 4

Index Branch：token 打分 → block max-pool → Top-k

Midx,(r)i,b = maxj∈B_b, j≤i ( Qidx,(r)i · Kidxj ) / √didx
I(r)i = TopKb( Midx,(r)i,· , k )　（local block 永遠納入）
  • 每個 GQA group 一個 index query head、所有 group 共享一個 index key head — 只多 兩個投影矩陣 Wqidx, Wkidx。
  • 先算 token-level score，再 block max-pool 聚合到 block 層級，每 query+group 取 Top-k block。
  • per-query 成本從 O(N) 降到 O(k·Bk)，固定不隨序列長度增長。
Main Branch 只對選中 block 內 causally-visible token 做標準 scaled-dot-product softmax。

方法 · 命題 2 / 4

訓練：KL alignment + 三道穩定機制

LKL = (1 / N·Hkv) Σi,r DKL( stopgrad(P(r)i,·) ∥ Pidx,(r)i,· )　·　L = LLM + λ Σlayers LKL
  • Top-k 不可微 → LM loss 訓不動 index 投影。用 KL loss 把 Index 分佈對齊 Main Branch teacher（只在選中 token 上算）。
  • Gradient Detach：對 Index 輸入 stop-gradient，KL 只更新 Wqidx/Wkidx、不流回 backbone。
  • Indexer Warmup：前期兩分支都跑 full attention、只用 KL 訓新投影，warmup 後才切 sparse。
  • Local Block：每個 query 的 local block 永遠被選，保留 immediate neighbourhood、防退化選擇。

方法 · 命題 3 / 4

計算複雜度：固定預算讓 gap 隨 N 拉開

FGQA(N) = 2·Hq·dh·N²
FMSA(N) = Hkv·didx·N²　+　4·Hq·dh·N·k·Bk
  • 第一項 = Index Branch（輕量打分），第二項 = Main Branch（固定 KV 預算 k·Bk，與 N 線性）。
  • 當 k·Bk ≪ N 且 Hkv·didx ≪ Hq·dh，FLOPs gap 隨 N 持續拉開。
  • 部署設定 Bk=128, k=16 → 每 query 只 attend k·Bk = 2,048 token。
dense 隨整個序列成長，MSA 把主注意力預算鎖死 → context 越長、相對省越多。

方法 · 命題 4 / 4

Kernel co-design：把理論稀疏變真加速

Exp-free TopK
  • softmax 保序 → 選 Top-k 不需 max/exp/sum，直接拿 raw score 排序。
  • warp 32 lane 各維護 k-element min-heap、shared-memory 無 bank conflict。
KV-outer iteration
  • 比較 Q-outer (FLOPs/IO≈G) vs KV-outer (≈⅔·Bk)；因 ⅔·Bk ≫ G，選 KV-outer + Q gather 最大化 arithmetic intensity。
  • two-phase forward + query concatenation 填滿 tensor core。
TopK 5.1× vs torch.topk3.7× vs TileLang radix

109B MoE，兩條訓練路線在同一 3T budget 對打

模型
  • 109B MoE（6B activated/token），41 層（3 dense + 38 MoE），128 routed + 1 shared expert、top-4。

  • MSA-CPT：從 2.6T full-attn checkpoint 接 400B token（40B warmup + sparse CPT）。

機制分析 · 皇冠頁

為什麼 MSA 能把「稀疏」訓到不掉能力

Top-k 不可微 → 純 LM gradient 給不了 indexer 有用信號。MSA 用 Main Branch 的注意力分佈當 teacher，靠 KL 教 indexer「哪些 block 在語意上該選」。
  • KL teacher 對齊：只在選中 token 上算 KL，把 Index 分佈 Pidx 拉向 Main Branch 的 P，indexer 學到的是「有意義的檢索」而非隨機。
  • Recall 全程高位（Figure 3b）：block recall（選中重要 block 比例）+ score recall（選中 block 佔 Main Branch 注意力質量比例）在整段 CPT 維持高位 → 可靠回收重要 block、且選中即抓住多數注意力質量。
  • 表徵自適應：native sparse pretraining（MSA-PT）讓模型表徵主動適應稀疏 pattern，多個 retrieval / 多模態 benchmark 反超 dense。

消融 1 / 2 · 三道「必要」機制

缺一不可：gradient 來源、detach、warmup

  • B.2 Index gradient 來源（Fig.7）：LM-only → 短文好但 long-context retrieval 差（top-k 無直接壓力）；KL-only → retrieval 好但短文退；LM+KL → 兩軸最佳平衡。
  • B.3 把 KL 限制在 Index Branch（Fig.8/9）：不 detach → 大 KL 係數下 gradient spike 傳進 backbone、幾百步內 LM loss 發散。根因是 self-distillation：backbone 靠「簡化自己的注意力分佈」來降 KL，而非改進 indexer。detach 後同係數穩定、短文退化消失。
  • B.4 Indexer warmup（Fig.10/11）：訓練最早期 Main Branch 注意力 entropy 急降（平滑→尖銳），稀疏選擇此時極脆。step-0 開 top-k → indexer 追快速移動目標、自身選擇近乎隨機 → 爛選擇把 Main Branch 導向無資訊 token。短 full-attn warmup 給乾淨初始化。
三者都是「讓不可微的稀疏選擇能穩定學起來」的支架，不是性能裝飾。
§B.2–B.4 p.24–26（Fig.7–11）
消融 2 / 2 · Occam's razor：拿掉什麼還能 work

最終 recipe 只強制「incomplete self block」，其餘交給模型自學

  • B.5 可學 attention sink（Fig.12/13）：GPT-OSS 式可學 sink logit 只吸走部分 sink 質量、無法消除 first-token sink，下游 PPL 無一致改善 → 不納入 recipe。
  • B.6 動態 vs sliding window（Fig.14）：FLOP-matched sliding-window（固定 first block + local window）在同預算下 PPL 全程高於 MSA → content-dependent 動態選擇勝過 position-fixed。
  • C.1 Block size（Table 4）：32 / 64 / 128 對品質影響有限 → 用大 block（128）換 kernel 效率。
  • C.2 強制 sink & local（Table 5）：拿掉強制選 first block + fixed local window 後，模型仍自發學出 sink 與 local 結構，reasoning / code / PPL / retrieval 幾乎不變。
  • C.3 Index value head（Table 6）：有 warmup 後拿掉 value head 無系統性退化 → 它只是「早期額外訓練信號」、非收斂期必要容量。inference 時 top-k 只需 block-wise max of Qidx·KidxT，完全避開 value aggregation 與 exp。

局限性 · 作者自承

殘留 retrieval gap、runtime 達不到理論、CPT 較保守

  • Long-context retrieval 殘留 gap：RULER-128K 的 CWE/FWE Δ−1.35、Rerank/RAG Δ−2.10 仍落後 Full；作者認為需更長 sparse 訓練 / 更大 inference 選擇預算 / 更豐富的 indexer scoring 來補。
  • Runtime < 理論 FLOPs 降幅：index 構建、top-k、reverse-index、gather、load-balance 開銷 + 不規則 memory access，讓 7.6×~28.4× 的理論值在 wall-clock 上打折（但隨 context 拉長持續擴大）。
  • MSA-CPT 較保守：部分 benchmark 反低於 dense（HumanEval 57.9 vs 61.0、EgoSchema 25.8 vs 29.6）—— 從既有 dense checkpoint 轉換時要接受這個取捨。
  • Outlook：把 selector-only 設計延伸到 RL post-training / agentic deployment 仍是 future work。
§5.3 · Outlook §7 p.13
個人解讀 · 實際應用場景

從 GQA checkpoint near-lossless 轉換，是最被低估的賣點

  • 不必從頭訓：MSA-CPT 證明既有 GQA dense 模型可用 ~400B token（含 40B warmup）轉成 sparse，而非重訓 3T —— 對「手上已有 dense backbone」的團隊是低成本長 context 升級路徑。
  • 固定注意力預算 = 可預測的部署成本：每 query 永遠只 attend k·Bk=2,048 token，無論 context 1K 或 1M，KV 計算上限鎖死 → latency / 顯存可預估，利於排程與 SLA。
  • Kernel 才是真價值：exp-free TopK（5.1× vs torch.topk）+ KV-outer 把理論稀疏變真加速 —— 提醒「sparse attention 的成敗在 kernel 工程，不只在演算法」。
演算法極簡（Occam's razor）+ kernel co-design，比堆三條並行分支更務實。

對圖像 / 多模態工程師的啟發

稀疏注意力對 native multimodal 不只無損，反而加分

  • 多模態反超尤其明顯：MSA-PT 在 VisualWebBench +12.8、EgoSchema +8.0、VideoMME +4.37 全面超越 dense baseline —— 對長 visual / video token 序列，content-dependent 稀疏選擇等於一種隱式的視覺 token 篩選，丟掉無資訊 patch 反而更乾淨。
  • 長 visual context 的 FLOPs 結構性省：高解析影像 / 多幀影片動輒數萬~數十萬 token，O(N²) 是 VLM 部署痛點；MSA 的固定預算讓「餵更高解析 / 更多幀」的邊際成本可控。
  • KL-teacher 思路可借鏡：用「精確分支當 teacher 教稀疏分支選什麼」的範式，可遷移到 image patch pruning / token merging 的可學選擇器設計。
  • block-level 動態選擇 > position-fixed：B.6 證明動態選擇贏固定 window —— 視覺任務的 ROI 本就 content-dependent，固定 pattern（如純 local window）會系統性丟關鍵區域。

本質貢獻 & self-distillation 失效模式

A lightweight Index Branch scores key–value blocks and independently selects a Top-k subset for each GQA group, enabling group-specific sparse retrieval while maintaining efficient block-level execution; the Main Branch then performs exact block-sparse attention over only the selected blocks.

輕量 Index Branch 對 KV block 打分、為每個 GQA group 獨立選 Top-k 子集，達成 group-specific 稀疏檢索同時維持 block 層級高效執行；Main Branch 隨後只對選中 block 做精確 block-sparse 注意力。Abstract p.1

We attribute this regression to a self-distillation effect: the backbone can lower the KL loss by simplifying the Main Branch attention distribution, rather than by improving the Index Branch.

我們把這種退化歸因於 self-distillation 效應——backbone 可以靠簡化 Main Branch 注意力分佈來降低 KL loss，而不是靠改進 Index Branch。（這正是為何 KL gradient 必須 detach、限制在 Index Branch 內）§B.3 p.25

附錄 · 螢光筆精華 2 / 2

Warmup 動機 · KV-outer 選擇 · prior 不需硬編

If top-k selection is enabled from step zero, the Index Branch must track a rapidly moving target while its own selections are still nearly random. Poor early selections then route the Main Branch to uninformative tokens.

若從第零步就開 top-k 選擇，Index Branch 必須追蹤一個快速移動的目標、而它自己的選擇還近乎隨機；早期的爛選擇於是把 Main Branch 導向無資訊的 token。§B.4 p.26

Since ⅔·Bk ≫ G in practice, we choose KV-outer iteration with Q gather to maximize arithmetic intensity.

因實務上 ⅔·Bk 遠大於 G，我們選擇 KV-outer 迭代搭配 Q gather 以最大化算術強度。§4.2 p.7–8

When the forced selection of the first block and the fixed local window are removed, the trained model still exhibits both structures: attention concentrates on the sequence prefix when useful, and nearby tokens remain frequently selected.

當移除對 first block 與固定 local window 的強制選擇後，訓練好的模型仍展現這兩種結構——有用時注意力會集中在序列前綴、鄰近 token 也仍頻繁被選。（故最終 recipe 不硬編這些 prior）§C.2 p.29
