MHLA: Restoring Expressivity of Linear Attention via Token-Level Multi-Head

Kewei Zhang*, Ye Huang*, Yufan Deng, Jincheng Yu, Junsong Chen, Huan Ling, Enze Xie, Daquan Zhou · Peking University / NVIDIA
ICLR 2026 PosterarXiv 2601.07832v22026-07-09

把 linear attention 的失準歸因到 global context collapse，再用 token-block multi-head mixing 把 query-conditioned selectivity 補回來。

Source: arXiv 2601.07832v2; OpenReview forum 340QjF3jJP (ICLR 2026 Poster).
TL;DR

Linear attention 快，但它把所有 token 壓進同一個 global KV summary；MHLA 的核心，是讓每個 query block 重新混合多個 local summaries。

一句話：不是多加一個補丁，而是把「查哪裡」這件事放回 linear attention。
Abstract p.1; §3.2 p.4-p.5; §4.1 p.5-p.6

為什麼這篇適合今天：週四 compute acceleration

  • 主題命中：針對 attention 的時間 / 記憶體瓶頸，從 O(N2) softmax 改成接近 O(N) 的 linear attention。
  • 不是純 benchmark trick：paper 先提出 failure mode，再用 rank / entropy 解釋為什麼一般 LA 會崩。
  • 工程落點清楚：標準 GEMM、chunkwise training、video 31,500-token setting，對長序列生成很實用。
  • 頂會優先：OpenReview search 顯示 ICLR 2026 Poster；ingest 以 arXiv PDF/raw 為事實來源。
§1 p.1-p.3; §4.2 p.7; §5.3 p.10

問題：softmax attention 太貴，linear attention 又太鈍

Softmax self-attention

  • 每個 query 對所有 key 做 pairwise similarity。
  • 時間 O(N2d)，記憶體 O(N2)。
  • 優點是 query-specific：每個 query 都能形成不同 token distribution。

Standard linear attention

  • 把所有 K,V 先壓縮成 global summary。
  • 時間可降到 O(Nd2) / O(Nd_phi)。
  • 代價是所有 query 共享同一個摘要，selectivity 變弱。
MHLA 的目標不是否定 linear attention，而是修掉「所有 query 讀同一個壓縮世界」這件事。
§3.1 p.4; §3.2 p.4

核心診斷：global context collapse

  • 標準 LA 的 global KV summary 是固定 d x d 容量；N 變大時，token-specific information 會被擠掉。
  • rank limitation：Alin = Q~K~T，所以 rank(Alin) <= d，當 N >> d 時必然是低秩近似。
  • loss of sparsity：每個 query 無法在 compression 前重加權 individual keys，attention 會趨向更均勻。
Global LA: G = Σj phi(Kj)TVj, then every query reads the same G.
§3.2 p.4-p.5

從一個 global summary，到多個 local summaries

Figure-first takeaway: 以前是所有 query 讀同一個 Global KV Summary；MHLA 讓 query block 讀加權後的 Local KV Summaries。

Figure 2 p.3

MHLA step 1：把 token 切成 M 個 block

  • 輸入 X 投影成 Q,K,V；採用 kernel feature map phi(.)。
  • 把 sequence 切成 M 個 non-overlapping blocks；vision 裡通常是 2D spatial grid，video 裡是 spatiotemporal grid。
  • 對每個 block b 建 local summary Sb 與 normalizer zb。
Sb = Σj in b phi(Kj)VjT,   zb = Σj in b phi(Kj)
§4.1 Eq.3 p.5

MHLA step 2：Multi-Head Mixing

這不是 transformer head 的 channel split，而是沿 token / spatial / temporal 維度切 block。名字有點容易誤會，這裡要看清楚呢。
Figure 4 p.6; §4.1 p.5-p.6

公式直覺：block selection x token reweighting

S~i = Σb=1..M mi,bSb = Σt=1..N mi,b(t) phi(Kt)VtT
  • mi,b(t)：query block i 對 token t 所在 block 的 block-level weight。
  • phi(q)Tphi(Kt)：block 內仍保留 kernel similarity，區分 individual tokens。
  • 所以 MHLA 的 selectivity 是兩段式：先選相關 blocks，再在 block 內選相關 tokens。
§4.2 Eq.5 p.7

我會怎麼看這篇：它的價值與風險

  1. 價值：把 efficient attention 從「替換 kernel」拉回「恢復 query-conditioned routing」這個更可解釋的層次。
  2. 價值：video long sequence 的實驗很有工程說服力，因為它真的碰到 softmax 太貴、vanilla LA 會崩的區域。
  3. 風險：M、block geometry、locality prior 都是新的設計自由度；換資料型態時未必免費。
  4. 風險：NLP 部分是支持性證據，不是壓倒性勝利；不要把 MHLA 解讀成所有 sequence modeling 的通用答案。
§4.1 p.5; §4.2 p.7; §5.3 p.10; §5.4 p.11

工程啟發：給圖像 / 影片生成系統的三個 takeaways

  • 如果你的 bottleneck 是高解析 / 長影片 attention，先問：是否真的需要每層保留 full softmax？
  • 若使用 linear attention 但品質掉很多，檢查它是不是把所有 token 壓成單一 global state。
  • 用 block/chunk geometry 表達空間或時間 locality，可能比堆額外 DWConv / gating 更乾淨。
也就是說，想省算力時別只看 Big-O；要看「壓縮後還能不能讓不同 query 找不同東西」。

Appendix · 原文螢光筆 1

“As the sequence length N increases, the information requiring representation exceeds the capacity of the fixed-size d x d matrix, leading to performance saturation. We term this phenomenon global context collapse.”

隨序列長度 N 增加，需要被表示的資訊量超過固定 d x d 矩陣容量，造成效能飽和；作者稱之為 global context collapse。§3.2 p.4

“Eq. 5 makes the mechanism transparent: each query-block rescales the contribution of entire blocks via m_i, and within each block the usual kernel inner product differentiates tokens.”

Eq.5 把機制說清楚：query block 先透過 m_i 重新加權整個 block，block 內再用 kernel inner product 區分 token。§4.2 p.7

Appendix · 原文螢光筆 2

“At this extreme sequence length, vanilla LA suffers severe degradation due to global context collapse, whereas MHLA preserves linear-time complexity and recovers performance comparable to the original FlashAttention-based Wan2.1-1.3B, achieving a 2.1x inference speedup.”

在極長序列下，vanilla LA 因 global context collapse 嚴重退化；MHLA 保留線性時間，同時接近原本 FlashAttention Wan2.1-1.3B 的表現，並達到 2.1x 推論加速。§5.3 p.10
