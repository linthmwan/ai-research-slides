Scaling Attention via Feature Sparsity

Yan Xie, Tiansheng Wen, Tangda Huang, Bo Chen, Chenyu You, Stefanie Jegelka, Yifei Wang
ICLR 2026 conference paperarXiv:2603.22300v2Attention / Kernel / Long Context

把注意力的稀疏性從 token 軸移到 feature 軸：保留全部 token pair，只讓 Q/K 啟動 Top-k feature。

Source: cover + abstract p.1; §1 p.2
TL;DR

SFA 不是少看 token；它是每一對 token 仍可互看，但只在少數 shared feature 上計分。

理論上把 QK 的 feature-dot 成本從 Θ(n2d) 壓到 Θ(n2k2/d)，再用 FlashSFA 避開 n x n score materialization。

1/64 theory @ d=128,k=162.5x speedup headline~41% KV-cache cut
Abstract p.1; §3.1 p.4; Fig.1 p.1
Problem

長 context 的瓶頸，不只在 token 數，也在每次 token pair 要算多少 feature

  • Dense attention 的核心成本是 O(n2d)：n 是序列長，d 是 head feature dimension。
  • 過去常見方向是 window / low-rank / kernel approximation / token sparsity；它們減 token 交互，但常犧牲 accuracy 或 retrieval。
  • 這篇的問題意識很直接：能不能保留高維 representation，只啟動少數最有用的 feature coordinate？
如果 token 軸已經被研究很多，那 feature 軸就是一條乾淨、可疊加的新加速軸。
§1 pp.2-3; §6 p.9
Paradigm Shift

Dense / token-sparse / feature-sparse 的差別

  • Dense: 全部 N x N token pair、每 pair 都用 d 維點積。
  • Token-sparse: 每個 query 只看一部分 key，但保留下來的 pair 仍用 d 維。
  • SFA: 全部 token pair 可保留，但 Q/K 先 Top-k，score 只來自 shared active feature。
Fig.2 p.3
Core Mechanism

SFA：先 Top-k，再只累加 support intersection

Q_tilde = TopKk(Q), K_tilde = TopKk(K)
sij = (1/sqrt(d)) Σu in Si ∩ Sj q_tildei,u k_tildej,u
  • 每個 query/key row 只保留絕對值最大的 k 個 coordinate。
  • score 不是 dense dot product，而是兩個 sparse support 的交集累加。
  • Backward 用 straight-through estimator：只讓 selected coordinate 回傳梯度。
§3.1 pp.3-4; Eq.3-Eq.6
Complexity Crown

Balanced support 下，非零 score edge 數 E 約等於 n2k2/d。

E ~= d * (nk/d)2 = n2k2/d
cost fraction ~= k2/d2
  • d=128,k=16：理論 feature-dot 成本約為 dense 的 1/64。
  • d=1024,k=32：比例約 1/1024，長 context 場景的想像空間很大。
  • 注意：這是 arithmetic model；真正 wall-clock 還要看 sparse kernel overhead。
§3.1 p.4
System Design

FlashSFA：把 sparse feature intersection 放進 FlashAttention 的 streaming pipeline

  • 沿用 FlashAttention 的 tiling 與 online softmax。
  • 把 dense tile matmul 換成 active-feature intersection。
  • score buffer 只在 tile 內存在，立刻餵給 online softmax。
關鍵是不寫出完整 n x n score matrix。
softmax(Q_tilde K_tildeT / sqrt(d))V
computed by tiled sparse overlaps
  • Q 用 CSR；K 用 feature-wise CSC posting list。
  • Binary search 把 feature posting list 限制到目前 key tile。
§3.2 pp.4-5; App. C.3-C.4 pp.17-18
Kernel Detail

沒有 atomicAdd：每個 score tile position 由固定 thread owner 累加

  • Grid = row tile x batch-head；一個 block 負責一段 row tile，常見 Br=128。
  • 8 warps 分別處理 disjoint row stripe；thread 被映射到 2 x 2 score patch。
  • 每個 S[r,c] 由唯一 thread 在 register 中累加所有 feature partial，避免 atomic contention。
  • V row 是 dense contiguous load；P@V 變成 sparse-dense accumulation。
這頁的重點：SFA 不是只有理論 sparsity，作者真的把 irregular access 拆到 CUDA execution model。
App. C.2-C.4 pp.17-18

Scaling Regime

越長、head dimension 越大，SFA 越不像小技巧，而像必要條件

  • 4k context 時收益存在但中等；65k context 時 sparse overhead 被 dense quadratic cost 淹沒。
  • 256-d head、65k context：作者報告 latency reduction 超過一個 order of magnitude。
Fig.4 p.7; §4.3 pp.7-8
Inference Memory

KV-cache 不是免費午餐：SFA 也在 memory axis 上省

  • 短 context <=4k：dense 仍可能競爭，因 sparse lookup 有 overhead。
  • 超過 8k-16k：SFA 開始穩定勝過 dense。
  • k=4 時 KV-cache saving 最高約 40%。
§4.3 p.8; Fig.5 p.7
Mem_CSR ~= N k (S_val + S_idx) + (N+1) S_ptr
ratio ~= 2d / (3k + 4)

作者在 Appendix J 用 fp16/bf16 value、int8 index、int32 indptr 推導 CSR memory ratio。

Pretrained Adaptation

直接把 dense-pretrained model TopK 化會有 distribution shift

L = LLM + λ · meanh || O_tildeh - stopgrad(Oh) ||2F
  • 作者承認 TopK 幾乎會重置原本 dense feature pattern。
  • 因此用 dense attention output 作 teacher-like regularization。
  • Qwen3-8B SFA k=16 在 NIAH 4k/8k/16k/32k 達 100/100/100/97，接近或高於 dense finetune。
  • GSM-8K 仍略輸 dense finetune，表示 reasoning 對 pruning 更敏感。
§5 pp.8-9; Table 3 pp.8-9
Mechanism

真正支撐 SFA 的不是「稀疏一定好」，而是 feature 使用沒有坍縮。

  • TopK index entropy: Q-head 0.88-0.98、平均 0.94；K-head 0.85-0.97、平均 0.93。
  • SVD: d=128 的 Q/K activation effective rank 約 50-60，說明 dense feature 有可壓縮冗餘。
  • 兩者合在一起：feature 可壓縮，但不能用短 embedding 粗暴壓扁。
App. F p.21; App. L p.26
Ablation

k 與 d_head 的 sweet spot：不是越稀疏越好

  • k >= 8 時 PPL 很接近 dense，但 32k latency 仍保有優勢。
  • d_head=64 是 paper 報告的折衷點：恢復大部分 dense performance，又保留加速。
App. H pp.23-24

Limits

這篇很紮實，但還不是「直接替換所有 attention」的結論

  • Sparse tensor products 仍需要更成熟的 GPU/CUDA library 支援，否則 theoretical FLOPs 不一定轉成 wall-clock。
  • 過小 k 會有品質退化；實務上需要 adaptive sparsity budget。
  • Token-level sparsity 與 dimension-level sparsity 如何最佳組合仍未解。
  • Dense-pretrained adaptation 有 distribution shift，不能期待無痛切換。
工程判讀：SFA 是長 context 新軸線，不是萬能壓縮器。
§7 p.10; §5 pp.8-9; App. H pp.23-24
Engineering Takeaways

如果要把 SFA 帶進真實模型，先看三個門檻

  1. Context regime: 目標是否真的在 8k-16k 以上？短 context 可能被 sparse overhead 抵消。
  2. Kernel maturity: 是否能用 feature-wise CSC / tiled online softmax / no-atomic accumulation 做到穩定吞吐？
  3. Training path: 從頭訓練最乾淨；pretrained retrofit 需要 output regularization 與任務回歸測試。
§4.3 p.8; §5 pp.8-9; App. C pp.16-18
Appendix Highlight 1

Instead of dense d-dimensional queries and keys, SFA learns k-sparse codes in which each token activates only a handful of coordinates.

SFA 不是砍 token，而是讓每個 token 只啟動少數 Q/K feature coordinate，保留 token 覆蓋率。§1 p.2

Appendix Highlight 2

The resulting scores are mathematically identical to computing softmax(Q_tilde K_tilde^T / sqrt(d))V, but with both compute and memory scaling as in SFA.

FlashSFA 的關鍵不是近似 softmax，而是用稀疏交集把同一個 SFA score 更省地算出來。§3.2 p.4

Appendix Highlight 3

For d = 128 and k = 16 ... the ratio is k^2/d^2 = 1/64.

這是 SFA 的核心槓桿：省的是每一對 token 的 feature-dot 成本，而不是只省 token 數。§3.1 p.4

Appendix Highlight 4

For the 16 Q-heads ... entropy ranges from 0.88 to 0.98 with an average of 0.94.

TopK 沒有明顯坍縮到少數 feature，這讓 sparse-feature 路線比 MoE 式硬 routing 更可信。App. F p.21
