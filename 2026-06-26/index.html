TL;DR

Ternary LLMs no longer have to be trained from scratch.

CAT-Q uses PTQ, not QAT: 512 calibration samples (~1M tokens) produce 1.58-bit LLMs that compete with BitNet / TriLM-style models trained on 100B–1T tokens.

Abstract p.1; §3.3 p.6-7
Problem

1.58-bit PTQ is attractive, but historically unstable

  • Ternary weights map each value to {-1, 0, +1}, giving >10x memory reduction vs FP16 and replacing many multiplications with additions/subtractions.
  • Standard PTQ is practical at 8-bit and sometimes 4-bit, but ultra-low-bit 2-bit / 1.58-bit settings suffer severe information loss.
  • Existing high-performing ternary LLMs mostly use QAT / retraining, which is expensive and architecture-specific.
CAT-Q asks: can ternary LLMs be made with the PTQ workflow engineers actually want?

§1 p.2; Abstract p.1
Core diagnosis

CAT-Q identifies two coupled failure modes

  1. Distributional misalignment: direct ternarization creates ternary weights whose distribution does not align with the pretrained FP weights.
  2. Non-differentiable convergence: hard ternarization is too abrupt for PTQ; unlike QAT, there is no long retraining phase to absorb the shock.
  3. Layer isolation: single-layer reconstruction ignores neighboring-layer compensation, worsening the error budget.
The method is built around these three points: LM for alignment, ST for convergence, sliding-window reconstruction for cross-layer compensation.

Method overview

CAT-Q learning flow: redistribute → soften → hard ternarize

Flow: W → Ŵ → differentiable ternarization → hard T; deploy as W ≈ αT.

Figure 1 p.3; §2.2 p.4
Ternary formulation

The target is still the classic ternary reconstruction problem

minα,T ||W - αT||22, where Ti ∈ {-1, 0, +1}
Ti = 1 if Wi > Δ; 0 if |Wi| ≤ Δ; -1 if Wi < -Δ
  • TWN and absmean-style ternarization estimate α and Δ from fixed statistics.
  • CAT-Q keeps the hardware-friendly target but learns how to approach it under PTQ.
§2.1 p.3 Eq.1-2
Learnable Modulation (LM)

LM does not just tune α and Δ — it changes the coordinate system

Ŵ = (W - µ) / α
µ = µ0 + δµα0
α = δαα0
Δ = δΔΔ0
  • µ0, α0, Δ0 are statistics-based priors.
  • δ factors are learned with calibration samples.
  • Ŵ is a redistribution proxy: the model learns a representation where ternary thresholding is less destructive.
  • Reconstruction remains W ≈ αT at deployment, preserving ternary implementation simplicity.

Softened Ternarization (ST)

ST turns hard discretization into a curriculum

§2.3 p.4-5 Eq.5-6
Sliding-layer optimization

The reconstruction target is a window output, not each layer alone

minA,T ||F(W, X) - F(A · T, X)||22
  • W is the set of FP weights in the current sliding window; A and T are the corresponding scales and ternary weights.
  • Neighboring layers become aware of each other, so compensation can happen inside the window.
  • This relaxes pure weight reconstruction into output reconstruction, reducing optimization difficulty.

Mechanism crown page

CAT-Q works because it separates “where to quantize” from “how to converge.”

  • LM finds a ternary-friendly coordinate system by learning distribution and threshold modulation.
  • ST makes the discrete jump gradual: identity → differentiable ternary → hard ternary.
  • Sliding reconstruction lets local layer windows compensate quantization error together.

Limitations

The method solves quantization, but the ecosystem is not done

  • Kernel support: optimized public 1.58-bit GPU/CPU kernels are still mostly tied to BitNet-style models; diverse ternary LLM kernels remain lacking.
  • Accuracy gap: there is still room to close the FP16 gap, especially on complex reasoning tasks.
  • MoE sensitivity: MoE models show larger quantization sensitivity, likely because fewer parameters are active per token.
  • Evaluation breadth: commonsense zero-shot is strong, but production instruction-following / long-context behavior needs more stress testing.
Appendix G p.16; §3.3 p.7
Engineer’s reading

What to take away if you build inference systems

  1. 1.58-bit PTQ is becoming less of a toy if the optimization path is designed for discreteness rather than borrowed from 4-bit PTQ.
  2. The bottleneck may shift from quantization algorithm to kernel availability and format standardization.
  3. For edge / local inference, ternary weight-only + low-bit activation paths are worth tracking, especially if llama.cpp-style formats mature.
CAT-Q is most compelling as a bridge: pretrained FP LLM → cheap PTQ → deployable ternary checkpoint.
