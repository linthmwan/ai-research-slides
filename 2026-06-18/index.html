PULSE: Training Acceleration for Large Diffusion Models with Automatic Pipeline Parallelism

Boran Sun*, Guoyong Jiang*, Lin Zhang, Chen Chen, Yuechen Tao, Zhishu Che, Jieling Yu, Shan Chang, Huaxi Gu, Fangming Liu, Bo Li
ICDCS 2026 acceptedarXiv 2606.19163 · announced 2026-06-17Diffusion training systems
把 UNet-style skip connections 從 pipeline parallelism 的負擔，變成 placement / scheduling 的顯式約束。
Source: arXiv:2606.19163 · §I p.1–2
TL;DR

PULSE 的本質不是「更聰明地傳 skip」，而是 讓大型 skip activation 不必跨 device 傳。

它把 skip-connected encoder/decoder layers colocate 到同一 device，用 local buffer 保存 activation，再用 wave schedule 與 PP/DP tuner 把吞吐補回來。

up to 2.3x throughputup to 89% comm reductionV100 + Ascend 910A
Abstract p.1; Conclusion p.11
Problem · diffusion 不是 decoder-only transformer

UNet 的長 skip connection 讓 1F1B 的 sequential assumption 破功

  • Diffusion / image-video backbones 常採 encoder-decoder + symmetric skip，用來保留 high-resolution detail。
  • Pipeline parallelism 假設 activation 只傳到相鄰 stage；UNet skip 會跨多個 stage。
  • Forward 傳 activation，backward 還要傳 gradient activation，P2P traffic 被放大。
§I–II p.1–3; Fig.1 p.1
Why this matters

在 naive PP 中，skip traffic 佔 inter-device communication 的 85.5%–95.0%。

這代表 bottleneck 不是抽象的「pipeline bubble」而已，而是 diffusion architecture 本身造成的 activation movement。

85.5%–95.0% skip traffic~30% ZeRO-3 time in comm
Fig.2–3 / §II-B–C p.2–3
Baseline failure mode

Sequential block-wise partition 會把一條 skip 變成多跳傳輸

  • 1F1B 將 encoder / decoder blocks 依序切到 devices。
  • 一個早期 encoder activation 要跨多個 pipeline boundary 才能抵達 decoder。
  • 形式化簡化下，traffic 隨 K 與 D 線性放大：((K+4)D/4 - 1)a。
§II-C p.3; Fig.4 p.3
Core idea

Skip locality：把 skip pair colocate，同 device 讀寫 local buffer

  1. Partition：對 symmetric encoder-decoder pairs 加 colocation constraint。
  2. Buffer：forward 將 skip activation 存本地，backward 本地取回。
  3. Schedule：用 wave-like execution 恢復 overlap，避免 colocate 後空轉。
  4. Tune：在 PP degree、DP replicas、microbatch size 間找最大 throughput。
§III p.4; Fig.5 p.4
Method 1 · partition objective

Balance stage runtime，但禁止破壞 skip symmetry

min maxm( TfS_m + λ · MaS_m / Binter )
  • 每個 operation 是 atomic unit；若 skip connects operation i 與 j，stage q 必須和 stage p-q+1 mirror。
  • λ 將 activation P2P cost 納入 stage bottleneck；Binter 取 scale-out bandwidth 的 worst case。
  • 目標不是最少 stages，而是在 feasible colocation 下最小化 slowest stage。
Eq.1 / §IV-B p.5
Method 1 · dynamic programming

Bidirectional DP 同時從 encoder prefix 與 decoder suffix 長出 partitions

  • dp(i,j,k)：用 k 個 partitions 覆蓋 prefix {l1...li} 與 suffix {lj...lop} 的最佳 bottleneck。
  • transition 同時選 left segment L(i',i) 與 right segment R(j,j')。
  • penalty c=∞ if any skip constraint is violated；直接把非法切法剪掉。
dp(i,j,k)=min max{ dp(i',j',k-2), L(i',i), R(j,j'), c(i',i,j,j') }
Implementation 把 naive O(pn4) 降到 O(pn3)，可部署到 hundreds of layers。

Partition effect

SDv2 的 block runtime heavy-tail，讓 balanced partition 很有價值

對 uniform transformer-like blocks，partition gain 會小；對 UNet down/up sampling 造成的 heterogeneity，gain 才會放大。
Fig.6–7 / §IV p.5–6
Method 2 · scheduler

Collocation 後，不能假裝仍是普通 1F1B

  • ILP variable xs,m,d,t assigns stage s / microbatch m to device d / time t。
  • Constraints：unique assignment、device exclusivity、fixed stage-device mapping、collocation equality、sequential order、monotonic microbatch order。
  • Objective：minimize Tmax；secondary heuristic anchors first stage to device 0 and improves locality/readability。
xs,m,d,t ∈ {0,1}; devices1 = devices2 for every collocated pair (s1,s2) ∈ C
Eq.6–13 / §V-A p.6
Method 2 · wave schedule

S=D 時退化成 1F1B；S=2D 時形成 mirrored wave

ILP 只在 small-scale offline 找 pattern，之後當 static template 複製到 larger deployment；不進 runtime critical path。
§V-B p.6–7; Fig.8–9
Method 3 · hybrid PP/DP tuner

最優點在 memory、microbatch、P2P traffic 的交界，不是 PP 越大越好

Tsample = Tsched / ( b · P · G ), subject to Mpeak < Mlimit
  • PULSE with PP degree P has 2P pipeline stages；peak memory 出現在 P 與 P+1 collocated stages。
  • Tsched 包含 slowest stage forward time、P2P communication、DP AllReduce。
  • Enumerate N=P·G and feasible b∈{1,2,4,...}，選 per-sample time 最低者。

Mechanism 皇冠頁

PULSE 的增益來自一個很乾淨的因果鏈：skip locality → less P2P traffic → larger feasible throughput。

它不是單靠 scheduler 把 bubble 填滿；真正的主因是把 long skip activation 從 network path 移到 local buffer path。

Scalability

固定硬體預算下，skip-aware PP 讓更大的 diffusion model 可以訓練

  • 2-node V100 model scaling：PULSE outperforms Hanayo and Megatron across model sizes。
  • UViT 6.0B：PULSE reports 57.1% throughput improvement。
  • SDv2 7.2B：PULSE enables training while other methods are out of memory。
對 image/video 生成工程師來說，這一頁的訊號是「同一批 GPU 能不能把更大的 denoiser 跑起來」。
Fig.12 / §VII-B p.9
Ablation 1 · partitioning

Dynamic partition 不是普遍魔法；它吃的是 architecture heterogeneity

  • SDv2：encoder/decoder down/up-sampling 導致 stage imbalance，dynamic partition 最多 +85.5% throughput (2.5B)。
  • UViT / Hunyuan-DiT：blocks 更 uniform，gain 只有 1–2%。
  • 因此 partitioner 的價值是「修 UNet 非均質 runtime」，不是單純重新排序。
若你的 diffusion backbone 越接近純 transformer stack，PULSE 的 main gain 會更偏向 skip locality / hybrid tuner，而非 DP partition。
Fig.13 / §VII-C p.9
Ablation 2 · hybrid parallelism

PP degree 有 sweet spot：SDv2 P=4 變快，P=8 反而被 communication 吃掉

  • SDv2 1.7B：P=2 只能 microbatch 16；P=4 允許 microbatch 32，throughput 186.2 → 257.0 samples/s。
  • P=8：inter-stage communication 過多，performance drops。
  • SDv2 comm volume：0.77MB at P=2 → 4.84MB at P=8；Hunyuan：13.6MB → 95.3MB。
Fig.14 / §VII-C p.9–10
Limits

這篇方法很漂亮，但適用範圍其實相當明確

  • Primary target：encoder-decoder architectures with structured skip symmetry。
  • Irregular / partial skip patterns 可能只能 selective collocation；其他 edges 仍需要 runtime communication。
  • ILP schedule 是 offline small-scale pattern discovery，不是 full-scale exact solve。
  • TP 未納入 search space；作者認為 TP 可在每個 PULSE stage 內組合，但 TP+PP co-optimization 留到 future work。
§V-B p.7; §VIII p.10
工程解讀

如果你在訓練 image/video diffusion，這篇最值得帶走的不是「又一個 PP scheduler」

  1. 先 profile：skip activation 是否真的是 communication 主體；若不是，PULSE 不是第一優先。
  2. 把 model graph 中高體積 skip pairs 標成 placement constraints，而不是讓 runtime 每次傳。
  3. 用 memory model 找 PP/DP/microbatch sweet spot；別把 PP degree 推到最大。
  4. 對 SDv2-like heterogeneous UNet，stage balancing 很重要；對 DiT-like uniform stack，別高估 partitioner gain。
