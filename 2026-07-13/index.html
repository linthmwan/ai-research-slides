Spinning Straw into Gold

把 LLM agent「失敗」rollout 裡其實完成的目標，重新標成可訓練的成功示範。

Zichao Li · Gang Wu · Zichao Wang · Ruiyi Zhang · Wanrong Zhu · Ryan A. Rossi · Vlad I. Morariu · Jihyung Kil
ICLR 2026 conference paperarXiv:2607.04235v1Hindsight Supervised Learning
Source: cover p.1; arXiv comment “Accepted to ICLR 2026”.
TL;DR

HSL 的核心不是「生成更多示範」，而是承認 agent 已經在探索中做對了一些事。

Auxiliary LLM 事後讀完整 trajectory，找出所有 actually achieved goals；target agent 再用這些 relabeled successful demonstrations 做 post-training。

Abstract p.1; §3 pp.4-6.
Problem

長視野 agent training 的浪費點：失敗 rollout 被整段丟掉

  • SFT 依賴 expert demonstrations；成本高、coverage 低，且不吃 agent 自己產生的經驗。
  • RL/DPO 類方法理論上能用 trial-and-error，但 sparse reward + long horizon 讓 high-return trajectory 很難先出現。
  • 在 ALFWorld/WebShop 這類 POMDP agent 任務中，一條 rollout 可能沒完成原 instruction，卻完成了別的 valid goal。
  • HSL 的切入點：把 unintended achievements 轉成 successful demonstrations，而不是只當 failure。
§1 pp.1-2; Fig.1 p.2.
Contribution

三個貢獻：relabel、mask/reweight、on-policy buffer

  1. Hindsight Supervised Learning: rollout collection → auxiliary-LLM relabeling → fine-tune on relabeled successes.
  2. All achieved goals: 不只 relabel failed trajectory 的 final achieved goal，也挖 intermediate / extra achievements。
  3. Action relevance: 對每個 relabeled goal 標出哪些 action 真的相關，訓練時 masking 掉 irrelevant detours。
  4. Theory-guided weighting: 用 coverage / optimality 觀點設計 demonstration reweighting。
§1 p.2; §3.2-3.4 pp.4-6.
Method overview

HSL 是加在既有 post-training 旁邊的一條 branch

  • Target agent 按原 instruction rollout；auxiliary LLM 事後讀 trajectory。
  • 每個 achieved goal 形成 `(I′, trajectory segment)`；dynamic buffer D′ 持續更新。
  • 可疊加在 SFT 或 DPO/ETO 上，而不是替代它們。
Fig.2 p.4; §3.2 pp.4-5.
Formalization

POMDP 裡，relabeling 比 solving 更容易

原任務

πθ(at | τt-1, ot, I)
  • agent 只能看到 partial observation
  • 要預測 action 對未知 environment dynamics 的長期效果

事後 relabel

{I′1,...,I′K} = Minst(τT, o*)
  • 完整 trajectory 已經在手上
  • 問題變成判斷「到底完成了哪些 goal」
這也是為什麼 Llama-3.3-70B 直接當 REACT agent 不夠好，但當 relabeler 反而能幫小 agent。
§3.1-3.3 pp.4-5; §4.2 p.7.
Relabeling

每條 trajectory 產出兩種 hindsight label

1 · Goal identification

  • 列出 rollout 中所有 valid achieved goals。
  • 若完全沒有 meaningful goal，丟棄該 trajectory。
  • 可以包含 intermediate achievements，不限最後狀態。

2 · Action relevance

  • 對每個 relabeled goal 標 `z_t ∈ {0,1}`。
  • `z_t=0` 的 action 不進 loss，避免模仿原 instruction 造成的 detour。
  • 最後存成 `(τ, I′, z)` 放入 D′。
§3.3 p.5.
Learning objective

mask + reweight：不要把 hindsight data 當乾淨 expert demo

wd = (nd/Td)α · nd
LD′ = E[ (1/T) Σt -zt log Pθ(at|τt-1,ot,I′) ]
LHSL = λLD′ + (1-λ)LDtrain
  • `n_d/T_d` favor efficient solution segments; `n_d` acts as a proxy for task difficulty.
  • 實驗設定：λ=0.3, α=0.8, D′ queue size=100, 每步 relabel b=18 trajectories。
Eq.(3)(4) §3.4 p.6; §4.1 p.7.
Mechanism crown page

Theorem 1 的訊息：HSL 會幫忙，前提是 hindsight expert 的 coverage 與 optimality 夠好。

Upper bound = ground-truth SFT KL + relabeled-demo KL scaled by κE←H + suboptimality δE。因此 on-policy refresh、masking、reweighting 是機制核心。

Theorem 1 Eq.(2) §3.4 p.6.

Mechanism from analysis

90% relabel accuracy 是 HSL 成立的經驗支點。

作者人工抽查 200 筆 relabeled demos，其中 180 筆正確；這支撐「hindsight relabeling 比直接執行任務容易」的假設。

180 / 200 correct90% zero-shot relabeling accuracy
§4.4 p.10; Fig.6 p.9.
Goal distribution

HSL 不是魔法：relabel buffer 覆蓋不到的 goal 還是弱

  • `put` 被 relabeled data 大量增強，agent 對 put 達到 perfect performance。
  • `examine` 在 relabeled set 低於 5%，成功率最低：54.85%。
  • `put-two` 比 `put` 困難，因為 put 是其 subtask；分布偏差會反映到 final policy。
這正對應 Theorem 1 的 κ coverage：沒被 hindsight expert 覆蓋，就很難學到。
§4.4 pp.9-10; Fig.5-6 p.9.
Failure mode

Relabeler 也會把「子任務完成」誤判成「完整目標完成」

  • 錯誤案例：agent cools a tomato，但沒有 put it in fridge；relabeler 卻標成完整 goal。
  • 這類錯誤會增加 δE，也解釋為什麼 action-level value/reward label 可能是下一步。
Fig.7 p.10; §4.4 p.10; Limitations p.16.
Where HSL works

適用條件：長視野、開放目標、多個可意外完成的子目標

  • 好場景：ALFWorld-style household tasks、web/GUI tasks with reusable subgoals、tool-use workflows。
  • 弱場景：WebShop-style one-shot purchase；每條 trajectory 幾乎只有一個 goal，hindsight 空間很小。
  • 必要條件：有可枚舉/可判斷的 goal space；relabeler 能事後可靠判斷 achievement。
  • 工程 tradeoff：training compute/token cost 增加，但 inference cost 不變。
§4.1 p.7; §4.2 p.8; Limitations pp.16-17.
Limitations

作者自承限制：最危險的不是 cost，而是 unintended goals 的價值與安全

  • 依賴強 external LLM 做 zero-shot relabeling；更複雜任務可能要微調 relabeler。
  • binary relevance label 太粗，無法分辨 relevant-but-suboptimal actions。
  • 仍需 ground-truth demonstrations 穩定訓練；沒有完全解決 exploration。
  • 真實世界的 unintended goals 可能低價值甚至 malicious，需要 utility scoring / safeguard filtering。
若把 HSL 用在真實 web/robot agent，relabeling 之前應先過 safety/utility gate。
Limitations pp.16-17.
Engineering read

對 AI 圖像 / 多模態工程的啟發

  1. 把 failed generations/trajectories 視為「未標註資料」，先問它們其實完成了什麼，而不是只丟掉。
  2. relabeler 的任務要盡量做成事後判斷，而不是即時控制；這通常更穩、更便宜。
  3. 別把 relabeled data 當乾淨 truth：需要 mask、weight、coverage diagnostics。
  4. 做 multimodal agent 時，unintended success 可能很多，但也更需要 safety filter。
Synthesis grounded in §3.3-3.4, §4.4, Limitations.

Appendix · Abstract highlight

“HSL is sample-efficient: on ALFWorld, it surpasses baselines trained on the full dataset while using only one quarter of the ground-truth demonstrations.”

HSL 的重點是 sample efficiency：在 ALFWorld，用不到四分之一 ground-truth demonstrations 就超過 full-data baseline。Abstract p.1

這句是整篇 paper 的 punchline，也是 deck 主線選擇 sample-efficiency 作為主結果的原因。

Appendix · Theory highlight

“The gap shrinks further as the occupancy coverage and the optimality of πH improve.”

當 hindsight expert 的 occupancy coverage 與 optimality 提升時，agent-expert gap 會進一步縮小。§3.4 p.6

Δ(θ) ≤ SFT-KL + CT κE←H √(1/2 · HSL-KL) + CTδE

這個 bound 解釋了 reweighting（coverage）與 masking（optimality）為什麼是必要元件。

Appendix · Analysis highlight

“Of these, 180 are correct, yielding an accuracy of 90%...”

200 筆人工抽查中 180 筆 relabel 正確，zero-shot relabel accuracy 達 90%。§4.4 p.10

如果 relabeler accuracy 明顯下降，HSL 的 δ_E 會變大，NoMask / weak base agent 的問題也會被放大。

Appendix · Safety highlight

“The unintended goals may have low value for the users or even be malicious.”

意外完成的目標不一定對使用者有價值，甚至可能是惡意目標。Limitations p.17

這是把 HSL 推到 open-world agents 前必須補上的 governance layer。
