SliderQuant: Accurate Post-Training Quantization for LLMs

Shigeng Wang, Chao Li, Yangyuxuan Kang, Jiawei Fan, Zhonghong Ou, Anbang Yao · Intel Labs China · BUPT
ICLR 2026 PosterarXiv:2603.25284v1Announced 2026-03-26
把 LLM PTQ 從「每層同等對待」改成 depth-aware sliding schedule：第一/最後層被更密集地協同量化，中間層維持均勻低成本節奏。
TL;DR

SliderQuant 的要點不是發明一個新 quantizer，而是把 量化窗口怎麼滑 變成模型深度敏感：淺層逐步擴張、深層逐步收縮、中間固定滑動，再用 intra-layer sliding 做增量共同量化。

在 W4A4 / W2A16 這種比較痛的設定下，這個 schedule 比 CBQ / OmniQuant 更穩；OpenReview 最後從 mixed scores 走到 ICLR 2026 Poster。

問題背景

主流 PTQ pipeline 暗含一個危險假設：所有 depth 一樣難量化

  • Layer-wise / block-wise / multi-block-wise PTQ 都是在模型深度方向上依序處理，只是切分粒度不同。
  • 這在 8-bit 或較溫和場景看起來合理；但到 W4A4 這類 aggressive setting，總量化誤差不再小。
  • 固定大小 sliding window 雖然有 overlap，仍把 shallow / intermediate / deep layers 用同一套 window size 與步長處理。
  • SliderQuant 的第一個動作，是先問：不同層對量化的 sensitivity 真的相同嗎？
§1 pp.1-2 · §3.1-3.2 pp.4-5
觀察 · Figure 1

敏感度不是平的：中間層較穩，第一/最後層最脆弱

  • Intermediate layers 通常比 shallow / deep layers 更容易量化。
  • 第一層與最後一層在各自區域中最敏感。
  • 隨著更多層被 sequentially quantized，量化 impact 會逐步放大。
Fig.1 / §1 p.2 lines 59-98
設計命題

如果第一/最後層最敏感，window schedule 就不該固定

  1. 對 shallow layers：讓第一層反覆出現在逐步擴張的 window 裡，建立 local-to-global synergy。
  2. 對 deep layers：讓最後一層反覆出現在逐步收縮的 window 裡，建立 global-to-local synergy。
  3. 對 intermediate layers：用 fixed-size sliding window 保持 even optimization frequency。
  4. 對每個 window 內部：再用 intra-layer sliding 把權重/activation 增量共同量化。
換句話說：SliderQuant 把「重複優化」集中在最怕量化的 depth，而不是全模型平均鋪開。

方法 · 基線形式

Fixed-size sliding 是很多 PTQ 的共同外殼

minimize ||F(W, X) - F(W_hat, X)||2 ；W/A quantization 時把 X 換成 X_hat
  • Window `{s, i}` 沿 layer direction 移動；兩個連續 windows 的 overlap 是 `s - i`。
  • 如果 `i = s` 沒有 overlap，就退化成 layer-wise / block-wise / multi-block-wise 的常見 sequential PTQ。
  • Window 變大通常準確率較好，但 calibration memory / compute 成本也會上升。
§3.1 p.4 lines 191-214 · Eq.1
方法 · Inter-layer sliding

三段 depth 對應三種滑動節奏

Shallow / first layer

  • PESW：window 從第 1 層開始，size 每步 +1。
  • 第一層始終參與，用 dense local-to-global synergy 保護最敏感的輸入端。

Intermediate

  • FSSW：固定 size，預設 `{s=2, i=1}`。
  • 用 overlap 與邊界設計維持 even optimization frequency。

Deep / last layer

  • PCSW：從所有 deep layers 開始，逐步收縮到最後一層。
  • 最後層始終參與，用 global-to-local synergy 保護輸出端。

Default

  • 作者依 ablation 設 `Ls=4`, `Ld=4`。
  • 目標是 accuracy-efficiency trade-off，而非最大 window 暴力堆。
§3.2 p.5 lines 280-301
方法 · Intra-layer sliding

每個 window 內再做增量式共同量化

N = 1 / gamma；default gamma = 0.5, N = 2
  • 對 inter-layer window 內的 `s` layers，沿 weight / activation dimensions 做 progressively expanded quantization。
  • Stage 1：共同量化前半部 weight/activation matrices。
  • Stage 2：共同量化整個 matrices，包含第一階段已量化部分。
  • 直覺：先讓局部參數對齊，再讓全 window 一起承擔 reconstruction error。
§3.2 p.5 lines 302-314
Quantizer wrapper

SliderQuant 不是孤立 quantizer：它接上 CS + LoRA + uniform quantizer

X_tilde_i = X_i / alpha_i； W_tilde_i = W_i * alpha_i + A_iB_i
  • `alpha_i` 是 channel-wise scaling：把 activation outlier 與 weight scale 做反向吸收。
  • `A_iB_i` 是低秩 LoRA refinement；paper 實驗中 rank `r=4`。
  • 最後用 uniform quantizer 量化 refined `X_tilde` / `W_tilde`，以便與既有 PTQ fair comparison。
  • 這讓貢獻焦點落在 schedule，而不是把所有效果歸給一個新 quantizer。

工程解讀

對圖像/多模態工程師，這篇的可借鏡點在 schedule 而不是 LLM 本身

  1. 先量測模組/深度敏感度，再決定壓縮策略；不要預設所有 block 同質。
  2. 固定 bit-width 不代表固定 schedule：即使每層 bit 相同，也可以調整校準窗口、重建頻率與 overlap。
  3. 把 expensive calibration cost 與 zero inference overhead 分開評估；部署端免費不代表整條 pipeline 免費。
  4. 審稿人問的 robustness 是實作前必做事項：小 calibration set、domain shift、schedule hyperparameters 都要 sweep。
Synthesis from §3-4 and OpenReview concerns
局限

漂亮的 low-bit 數字之外，仍有四個風險

  • 新意邊界：layer sensitivity 與 sliding window 都有先例，貢獻主要是組合成 depth-aware PTQ schedule。
  • 校準成本：多窗口、多階段會增加 quantization-time compute / memory；paper 主文沒有完整成本表。
  • 超參數：`Ls`, `Ld`, `s`, `gamma` 都可能依模型而變，是否能自動從 sensitivity estimate 推出仍未解。
  • 因果拆解：adaptive schedule 與 repeated optimization frequency 糾纏，還需要更乾淨的 ablation 拆開。
OpenReview weaknesses/questions · §4.2
Appendix highlight · 原文 / 中譯

“intermediate layers usually have smaller quantization impacts on model accuracy compared to shallow/deep layers… the first/last layer has the largest quantization impact…”

中間層相對好量化，淺層/深層較敏感，而第一層與最後一層最敏感。§1 p.2 lines 86-98

“With PESW, the first layer is always involved in the quantization process with every expanded sliding window…”

PESW 讓第一層反覆參與逐步擴張的窗口，替淺層建立由局部到全域的量化協同。§3.2 p.5 lines 280-287

Appendix highlight · 原文 / 中譯

“In the first stage, the first half of weight/activation matrices… In the second stage, the whole… matrices… are quantized jointly.”

intra-layer sliding 用兩階段（預設）把每個 window 內的權重/activation 從半量到全量共同量化。§3.2 p.5 lines 307-314

“Under the W4A16 setting, SliderQuant remains near-lossless relative to FP16 on both 14B and 32B models…”

在 DeepSeek-R1 distilled 14B/32B 上，W4A16 幾乎貼近 FP16，這比只看 perplexity 更能說明推理能力保存。§4.1 p.8 lines 1357-1362

Takeaway

量化不只是 bit-width 選擇，也是 校準與重建的路徑設計。SliderQuant 的價值，是把 layer sensitivity 轉成一個可實作的 sliding schedule。
