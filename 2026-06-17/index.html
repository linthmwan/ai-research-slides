STARFlow-V：用 Normalizing Flow 端到端做影片生成

Jiatao Gu · Ying Shen · Tianrong Chen · Laurent Dinh · Yuyang Wang · Miguel Ángel Bautista · David Berthelot · Josh Susskind · Shuangfei Zhai —— Apple
CVPR 2026 · AcceptedarXiv 2511.20462v2 · cs.CVAnnounced 2025-11-27
第一個達到高品質自回歸文生影片的 Normalizing Flow 模型——挑戰 diffusion 在影片領域近乎壟斷的地位。Code: github.com/apple/ml-starflow
TL;DR

把可逆、likelihood-based 的 NF 重新帶回影片：一個可逆主幹原生支援 T2V / I2V / V2V，靠 global–local 架構、flow-score matching、block-wise Jacobi 推論（~15× 加速）三招，在 unimodal latent 上做時序推理，結構性繞開 AR diffusion 的 exposure-bias 累積誤差。

Abstract, p1；Intro p1–3
問題 · 前人缺陷

Diffusion 的非因果、非端到端，傷害了 AR 影片生成

  • 非端到端訓練：diffusion 對 frame 注入隨機等級的噪聲、學單步 denoiser，每次更新只監督一個噪聲等級 → 成本高，對影片尤甚。
  • 本質非因果：denoising 過程中未來 frame 會反過來影響較早的 frame，對串流 / 互動式使用不自然。
  • Exposure bias：AR diffusion 變體加 frame-wise 噪聲排程，但訓練吃 ground-truth context、推論吃自己不完美的預測 → train–test 落差隨時間累積，長 horizon 品質崩壞。
§2.1–2.2, p3–4
前提 · 為什麼選 NF

Normalizing Flow 是天生合適的工具

L = E[ log p₀(fθ(x)) + log|det Jfθ(x)| ]　　取樣 = 一次反演 x = fθ⁻¹(z)，無需迭代去噪
  • NF 學一個可逆雙射 fθ: x→z，用精確最大似然（change-of-variables）訓練；取樣只需一次反演，沒有迭代 denoising。
  • STARFlow（Gu et al. 2025，影像）已證明 Transformer 參數化的自回歸 flow 在 text-to-image 上可與 SOTA diffusion 競爭——STARFlow-V 把它延伸到影片，是 VideoFlow（2019，低解析、領域受限）以來第一個。
  • 每個 AF block：z = [x − μθ(x⊙m)] / σθ(x⊙m)，配 causal mask m；T≥3 個 block 交替 L→R / R→L 即可做通用密度建模。
§2.2, Eq 2.1–2.2, p4–5
核心貢獻 · 總覽

四個關鍵創新

  1. Global–Local 架構：deep flow 因果地傳播全域時序、local flow 限制在 frame 內 → 同時拿到可逆性、因果性、端到端 MLE 與可串流。
  2. Flow-Score Matching (FSM)：訓練一個輕量因果 neural denoiser 去回歸模型自身的 score，壓掉高頻 speckle 又不破壞因果性。
  3. Block-wise Jacobi 推論：把 AF 反演重寫成不動點系統、塊內並行塊間串行，配 video-aware 暖啟動 → ~15× 降延遲。
  4. 一個主幹三種任務：靠可逆性，同一主幹不改架構即原生支援 T2V / I2V / V2V。
§3.1–3.4, p5–8

🔑 皇冠機制 · 貢獻 1

Global–Local 架構：在 latent 上做時序，而非在 pixel 上

pθ(x) = ∏n=1..N pD(un | u<n) · |det JfS(xn)|　，　un = fS(xn)
  • Deep–shallow 分解 fθ = fD ∘ fS。天真移植會非因果，因為 fS 會把資訊從未來 frame 傳回過去。
  • 修正：把 fS 限制在每個 frame 內運作；只有 deep flow fD 因果地傳播全域時序脈絡。
  • 改寫成 frame 上的 AR 分解後，pD 成為 latent 空間裡的高斯 next-token 預測——把影片變成「連續語言模型」。
關鍵洞見：在 unimodal 的 latent u 上做時序推理，而非多模態的 pixel——這正是它能撐到 30s 的結構性原因。
§3.1, Eq 3.2, p5–6
貢獻 1 · 四個好處

Global–Local 一次給齊四個性質

表達力 & 穩健性

  • (a) 通用性：local stack fS 仍實現 per-pixel 無限高斯混合，表達力不損。
  • (b) 穩健性：取樣只條件在先前生成的 latent（非 pixel）上 → data-space 誤差不傳播，緩解 AR diffusion 的累積誤差；可逆 u↔x 避免資訊損失。

訓練 & 串流

  • (c) 端到端訓練：整個模型是一個 NF，用精確 MLE 訓練，無 per-step denoising 排程 / 代理損失 → train–test 落差更小。
  • (d) 可串流：fD⁻¹ 因果地逐 token/frame 取樣 un，fS⁻¹ 各 frame 獨立解碼 → 後面的 frame 無法影響前面的。
§3.1, p6
方法 · 貢獻 2

Flow-Score Matching：因果地去掉 NF 的訓練噪聲

Ldenoise(ϕ) = E‖ sϕ(x̃) − σ∇x̃ log pθ(x̃) ‖²　，　x̃ = x + ε，ε~N(0,σ²I)
  • NF 訓練需注入小噪聲（σ-smoothed 密度 qσ）以穩定（TARFlow），所以輸出略帶噪、需要一個 denoiser。
  • 兩個失敗選項：(a) Decoder fine-tuning 在 3D causal VAE 上失敗、丟失時序一致性；(b) Score-based denoising（Tweedie）有兩問題——噪聲梯度造成大動作區亮點 speckle、且 score 本質全域非因果（依賴未來 frame），破壞可串流。
  • FSM 解法：訓練輕量因果 denoiser sϕ 回歸模型的 score。NN 平滑壓掉高頻 speckle；因果性烤進 sϕ（一 frame look-ahead、全域因果、單步延遲）。
嚴格因果（≤n）反而失敗——時序差分對判斷去噪方向至關重要。靠快取 fθ 反傳梯度當 target，聯合訓練、開銷極小。
§3.2, Eq 3.3–3.4, p6–7
方法 · 貢獻 3

Block-wise Jacobi：把 AR 反演從 >30 分鐘壓到實用

x = μθ(x⊙m) + σθ(x⊙m)·z　（不動點系統，三角結構保證收斂）　殘差判停：‖x⁽ᵏ⁺¹⁾−x⁽ᵏ⁾‖² / ‖x⁽ᵏ⁺¹⁾‖² < τ，τ=0.001
  • 天真 AR 取樣慢到離譜：5s 480p@16fps 影片用 3B 模型要 >30 分鐘，遠非即時。
  • 把 AF 反演重寫成不動點系統、用並行 Jacobi 迭代解；把 token 切成大小 B 的 block：塊間串行、塊內並行，已完成的 block 快取為 KV context。
  • Video-aware 初始化：用前一個已收斂的 frame x⁽ᵏ⁾n 暖啟動下一 frame；時序連貫提供有效暖啟。預設非對稱 block size：首 frame 64、之後 512。
結果：≈15× 降推論延遲且維持保真。另加 pipelined decoding：fD 連續跑、fS 執行緒並行消化/精煉/去噪，延遲由 deep block fD 主導。
§3.3, Eq 3.5, p7

貢獻 4 · 一個主幹三種任務

可逆性帶來的多功能：不改架構，只換 conditioning

  • I2V：把首個觀測 frame 正向 flow-encode 種入 KV cache，再生成其餘——無需獨立 encoder。
  • V2V：把所有來源 frame flow-encode 填 KV，在任務線索（in/outpaint mask、編輯文字、相機/姿態）下 roll out；未編輯區直接複製、只合成編輯處。
  • 超出訓練長度：fD 內滑動視窗（chunk-to-chunk），重跑最後 Δ 重疊 latent 重建 KV；訓練時隨機丟首 frame 模擬 restart → 串流。
§3.4, p8

🔑 機制詮釋 · 為什麼 work

因為時序推理發生在 unimodal 的 latent，而非多模態 pixel：每一步是高斯 next-token 預測、條件在先前生成的 latent 而非 pixel——data-space 誤差無從傳播，結構性繞開 AR diffusion 的 exposure-bias 累積。這正是它撐到 30s 而 baseline 崩壞的真因。

§3.1(b) + §4.3；綜合機制讀解
局限 · 誠實面對

三個現實限制與展望

  • (1) 延遲：即便 ~15× 加速，在 commodity GPU 上仍遠非即時。
  • (2) 資料品質與 scaling：受限於資料集噪聲/偏差；當前 curation 下未觀察到乾淨的 scaling law。
  • (3) 非物理生成：仍有許多不合理輸出——章魚穿過玻璃罐壁、岩石在落地山羊腳下憑空出現。
未來：更快的取樣排程、蒸餾/剪枝成精簡 student、針對大動作與物理場景的主動資料選取。
§5, p10–11
個人解讀 · 影像工程師視角

這是一場範式賭注，不是排行榜冠軍

  • 用幾個 VBench 點數，換 diffusion 難以乾淨提供的性質：精確 likelihood、端到端 MLE、真正的因果串流，以及一個兼任自身 encoder（I2V/V2V）的可逆主幹。
  • Global–local（latent AR + frame 內 local refinement）是關鍵洞見：在 unimodal latent 上做時序推理，結構性避開困擾 AR diffusion 的 exposure-bias 累積。
  • 對做 world model / 互動式影片的人：「可逆 + 因果 + 精確似然」是一個有意義地不同的工具箱——值得納入長序列、可控、串流生成的設計考量。
Setup 備忘：70M text-video（Panda-70M HQ + in-house）+ 400M text-image；WAN2.2 的 3D Causal VAE（×16 空間/×4 時序/48-ch latent）；7B 參數；96× H100。

關鍵原文摘錄（一）：global–local 的因果修正

A naïve port of the deep–shallow flow is non-causal, because the shallow flow fS propagates information from future frames to past ones. We restrict fS to operate within each frame; only the deep flow fD propagates global temporal context causally.

深淺 flow 的天真移植是非因果的，因為 shallow flow fS 會把資訊從未來 frame 傳到過去。我們把 fS 限制在每個 frame 內運作；只有 deep flow fD 因果地傳播全域時序脈絡。§3.1, p5–6

Sampling conditions on previously generated latents, not pixels, so data-space errors do not propagate — this mitigates the compounding error of autoregressive diffusion.

取樣條件在先前生成的 latent 而非 pixel，因此 data-space 誤差不會傳播——這緩解了自回歸 diffusion 的累積誤差。§3.1(b), p6

附錄 · 中英對照重點

關鍵原文摘錄（二）：FSM 與快速推論

We train a lightweight causal neural denoiser sϕ to regress the model's score. The smoothness of the network suppresses high-frequency speckle, while causality is baked into sϕ with a one-frame look-ahead, keeping the system globally causal at one-step latency.

我們訓練一個輕量因果 neural denoiser sϕ 去回歸模型的 score。網路的平滑性壓掉高頻 speckle，而因果性被烤進 sϕ（一 frame look-ahead），讓系統在單步延遲下保持全域因果。§3.2, p6–7

We recast the autoregressive flow inversion as a fixed-point system solved by parallel Jacobi iteration: sequential across blocks, parallel within. Combined with video-aware initialization, this yields roughly 15× lower inference latency.

我們把自回歸 flow 反演重寫成不動點系統，用並行 Jacobi 迭代求解：塊間串行、塊內並行。結合 video-aware 初始化，得到約 15× 更低的推論延遲。§3.3, p7
