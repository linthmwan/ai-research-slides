In-Place Test-Time Training

把 MLP 的 Wdown 變成推論時可更新的 fast weights，讓既有 LLM 不改架構也能做 test-time adaptation。

Guhao Feng, Shengjie Luo, Kai Hua, Ge Zhang, Di He, Wenhao Huang, Tianle Cai · ByteDance Seed / Peking University
ICLR 2026 OralarXiv 2604.06169v1Announced 2026-04-08
來源: title page p.1; OpenReview decision Accept (Oral).
TL;DR

Test-Time Training 不一定要重做架構；也可以把既有 MLP 投影矩陣原地變成動態記憶。

In-Place TTT 的關鍵不是「多一個 memory layer」，而是讓 pretrained LLM 內部最大、最常見的 MLP matrix 同時承擔 slow knowledge 與 fast context。

Abstract p.1; §3.1 p.4.
Problem

靜態 LLM 的瓶頸：context 在變，但權重不會變

  • train-then-deploy 讓模型推論時無法把 continuous stream 內化進權重。
  • In-context learning 只能把 past tokens 留在有限 window；attention 又有 quadratic complexity。
  • TTT 理念上可更新 fast weights，但過去方法對 LLM 有架構、效率、目標三重不相容。
作者要解的是：如何讓 TTT 變成 pretrained LLM 可以接受的「增強模組」，而不是另一個要重訓的新架構。
Introduction p.1-2; Preliminary p.3.
Prior TTT barriers

三個阻礙讓 TTT 很難直接進入 LLM ecosystem

  1. Architectural compatibility: 既有 TTT 常新增/替換 layer，pretrained checkpoint 難 warm-start。
  2. Computational efficiency: per-token fast-weight update 太 sequential，無法吃滿 GPU/TPU parallelism。
  3. Objective mismatch: reconstruction target 只記當前 token representation，不一定幫助 next-token prediction。
Introduction p.2; Desiderata p.3.
Core contribution

一個「原地」設計同時避開三個坑

  • 不替換 attention；保留原本 Transformer token mixer。
  • 把 gated MLP 的 final projection Wdown 作為 fast weights。
  • 用 chunk-wise update 取代 token-by-token update。
  • 把 target 改成帶 future-token signal 的 LM-aligned objective。
  • 用 prefix scan 做 context-parallel computation。
  • OpenReview 最終為 ICLR 2026 Oral。
§3.1-3.4 p.4-7; OpenReview reviewer JSON.

Fast weights

為什麼選 MLP 的 Wdown？

O = (φ(HWgateT) ⊙ (HWupT)) WdownT
  • Wup 與 Wgate 保持 frozen slow weights；只讓 Wdown 在 test time 更新。
  • MLP 已被視為 key-value memory；作者把這個 slow memory 擴展成可吸收 transient context 的 fast memory。
  • 這是 drop-in：不新增 randomly initialized layer，也不破壞 pretrained attention。
§3.1 p.4.
Chunk-wise update

把 sequential TTT 改成可平行的大 chunk 更新

Apply: O[i] = Z[i](Wdown(i))T
Update: Wdown(i+1) = Wdown(i) - η ∇WL(Z[i](Wdown(i))T, V[i])
  • chunk size C 同時控制 fast-weight 更新顆粒度與硬體 parallelism。
  • 因為 attention 沒被拿掉，TTT 不必承擔 primary token mixer；可以用較大 chunk。
  • Ablation: C=512 / C=1024 表現較佳，C=1024 兼具效率。
§3.1 p.4-5; Fig.3 / §4.3 p.10.
LM-aligned objective

從「重建現在」改成「預測未來」

V_hat = Conv1D(X0) Wtarget
L(·,·) = -<·,·>F
Wdown(i) = Wdown(i-1) + η V_hat[i]TZ[i]
  • Conv1D 讓 target 帶 localized future-token information。
  • next-token target 可視為特殊情況：kernel 只指向下一個 token。
  • 加性 update 形式使 prefix-scan context parallelism 成為可能。
§3.2 p.5; Eq.(1).
Mechanism crown page

這篇最重要的機制命題：fast weights 應該儲存「對未來 token 有用」的資訊，而不是複製現在。

在 induction-head setting 下，LM-aligned target 期望提高 correct next-token logit；reconstruction target 對 correct token 只有 negligible effect。

Theorem 1 §3.3 p.6.
Theorem intuition

Logit-wise effect：LM target 直接推正確 next token

Correct logit: E[Δln[v*]] ≥ λlr · cnorm2 · calign
Other logits: |E[Δln[w]]| ≤ λlr · ε · calign
  • 若 key k* 在後文再出現，fast weights 應把它連到下一個 token v*。
  • LM-aligned target 用 Ex(t+1) 更新，因此 query reappears 時能推高 v*。
  • reconstruction target 用 current token embedding，只會記 k* 本身，對 v* 幫助不足。
§3.3 p.5-6.
Context parallelism

Prefix scan 讓 causal fast weights 可以平行算

  1. 所有 chunks 平行計算 activation Z[i] 與 update delta ΔWdown(i)。
  2. 對 delta 做 prefix sum，取得每個 chunk 開始前應看到的 fast-weight state。
  3. 用該 state 平行計算每個 chunk 的 output；causal padding 與 document-boundary reset 防止 future / cross-document leakage。
它不是「假裝平行」：作者明確聲稱數學上等價於 strict causal sequential process。
§3.4 p.6-7.

Ablation

三個設計選擇都不是裝飾

  • State size: state 越大，performance 越好；支撐「MLP fast memory capacity」敘事。
  • Chunk size: C=512 / 1024 較佳，C=1024 效率更好。
  • Conv1D: 對 long context 特別重要。
  • Wtarget projection: 對 short context 重要；兩者合起來才完整。

Limitations

要小心讀的地方

  • 雖然補了 8B / 14B，證據仍主要在 language-only、non-frontier scale；VLM / MoE / 更大模型還未充分驗證。
  • Wdown 選擇合理但未 exhaustive；attention output projection 或多矩陣組合可能是後續研究。
  • Figure 4 未在 raw text 中表列精確 TPS/GB，deck 不應自行讀圖估數。
  • TTT generation 的 causality 很敏感：實作必須正確處理 chunk boundary、causal padding、document reset。
§3.4 p.7; §4.1-4.3 p.8-10; reviewer concerns.
Engineering interpretation

如果你在做 LLM systems，這篇的啟發是什麼？

  1. 把「動態記憶」放進既有大矩陣，比新增一個小 module 更容易保留 pretrained compatibility。
  2. fast weights 的 objective 要對任務 aligned；對 LM 來說，記 current token 不如記 predictive signal。
  3. chunk-wise + prefix scan 是這篇從想法走到可部署的關鍵；否則 TTT 會停在漂亮但慢的 online update。
  4. 長 context 不只靠 attention pattern；也可以讓模型把 context 壓進可更新參數。

Drop-in design

“In-Place TTT treats the final projection matrix of the ubiquitous MLP blocks as its adaptable fast weights, enabling a ‘drop-in’ enhancement for LLMs without costly retraining from scratch.”

In-Place TTT 把常見 MLP block 的最後投影矩陣當作可調 fast weights，因此可作為不需從頭訓練的 drop-in 增強。Abstract p.1

Why LM-aligned objective matters

“The LM-Aligned target is guaranteed in expectation to increase the logit of the correct next token ... In contrast, the reconstruction target provides no such predictive benefit.”

LM-aligned target 在期望上會提高正確 next token 的 logit；相反地，reconstruction target 沒有這種預測收益。Theorem 1 discussion, §3.3 p.6

Takeaway

In-Place TTT 把 LLM 的 MLP 從靜態 feed-forward 模組，重新詮釋成可在推論時吸收 context 的高容量 fast memory。
