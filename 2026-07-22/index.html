Accelerating Autoregressive Video Diffusion via History-Guided Cache and Residual Correction

Kepan Nan, Wangbo Zhao, Penghao Zhou, Jun Li, Zhenheng Yang, Jian Yang, Ying Tai
CVPR 2026 Poster2026-07-22 TWARDM acceleration

ARCache: 用 history-aware cache refresh 與 clean residual reference，讓長影片 ARDM 加速時不把 early approximation error 一路放大。

Source: CVF Open Access PDF, cover / Abstract p.1
TL;DR

Caching video ARDMs is not a denoising-step problem; it is a history contamination problem.

ARCache 將問題拆成兩個控制點：HGC 決定何時安全 reuse；ERC 決定如何不用污染軌跡修正 residual。

Abstract p.1; §3 p.3-5
Problem

Why SDM caching breaks on autoregressive video

  • SDM 通常一次生成固定長度序列；ARDM 逐 frame / segment 生成，能支援 longer horizon 與更細 temporal control。
  • 既有 feature caching 利用 denoising timesteps 的 redundancy；在 SDM 裡可直接 reuse intermediate features。
  • 但 ARDM 每段都 conditioned on previous generated history；early cache approximation error 會進入 history loop，逐段累積成 artifacts。
  • 所以 acceleration 的瓶頸不是「能不能少算」，而是「少算造成的錯會不會被後續片段放大」。
§1 p.1-2; §2.1-2.3 p.2-3
Contribution map

ARCache targets two separate error paths

  1. History-Guided Cache (HGC): monitor history-token variation, not full-input drift, to trigger cache refresh.
  2. Enhanced Residual Correction (ERC): use first-segment clean residual trajectory to correct later segments.
  3. Training-free integration: designed for existing autoregressive video diffusion / world model pipelines.
  4. 3-model validation: FramePack-F1, SkyReels-V2, Matrix-Game show 2.88x / 1.87x / 3.13x fast-mode speedups.
Abstract p.1; §3 p.3; Table 1 p.7
Failure mode

The same cache reuse has different consequences in SDM vs ARDM

  • SDM + cache: all frames are generated jointly; cache error remains local to the denoising pass.
  • ARDM + cache: segment N is conditioned on generated segment N-1; approximation error becomes historical context.
  • ARDM + ARCache: suppress intra-segment error and mitigate inter-segment propagation.
Long-horizon quality is governed by error propagation, not only per-step approximation.
Fig.1 p.1; §1 p.2
Baseline math

Naive caching assumes adjacent denoising states are close

SDM cache: F(xt-k) := F(xt),   k in {1,...,N}
ARDM naive cache: F([hst-k, xst-k]) := F([hst, xst])
  • In theory, reuse for N later steps yields an `(N+1)`-fold acceleration opportunity.
  • In ARDMs, `h` is generated history; if `h` carries earlier cache errors, later segments inherit and amplify them.
  • This is why a paradigm-agnostic cache schedule is insufficient for video ARDM serving.
Eq.(1)-(2), §3.1 p.3-4
System overview

ARCache adds one scheduler and one residual-correction path

Mechanism 1 · HGC

Cache refresh should follow history-token change, because history best predicts ARDM output transitions.

Figure 3 reports that History-Output correlation dominates Segment-Output / full-input alternatives on FramePack-F1.

§3.2 p.4; Fig.3 p.4
HGC evidence

History is the most reliable cache signal

  • ARDM layer input = history tokens `hst` + current noisy segment tokens `xst`.
  • Paper measures adjacent-step input-output correlation with Spearman correlation and P-value.
  • History variation is more predictive of output transition, so HGC watches history deviation rather than global model input.
This is the key domain-specific change versus TeaCache-style full-input monitoring.
Fig.3 p.4; §3.2 p.4
HGC rule

A single threshold delta becomes the speed-quality knob

Delta(hst) = ||hst - hst+1||1 / ||hst+1||1
Keep cache while accumulated Delta <= delta; refresh when the cumulative deviation crosses delta.
  • Small `delta`: more recomputation, better fidelity.
  • Large `delta`: more feature reuse, faster generation, higher approximation risk.
  • Interpretation: unchanged history means stale features are probably safe; changed history means new context must be recomputed.
Eq.(3)-(4), §3.2 p.4-5
Mechanism 2 · ERC

Do not correct a drifting segment by following its own drift.

ERC anchors later residual correction to the first segment, whose trajectory is not contaminated by historical cache errors.

§3.3 p.5; Fig.4 p.5
Residual trajectory

First-segment residuals act as a clean reference

  • TaylorSeer-style layer-wise correction is memory-heavy and can OOM on long ARDM sequences.
  • Paper observes stable residual trajectories across unaccelerated segments via PCA.
  • Later accelerated segments have contaminated trajectories; using them for correction can amplify drift.
  • ERC reuses the first segment’s trajectory parameter as the reference for all later segments.
Fig.4 p.5; §3.3 p.5
ERC formula

Replace segment-specific trajectory parameter with first-segment parameter

rst = rstb + lambdast(rsta - rstb)
For s > 1, use lambdast := lambda1t = L1rel(r1t, r1tb) / L1rel(r1ta, r1tb)
  • `ta` and `tb` are recent recomputation timesteps.
  • The clean first segment avoids using corrupted later-segment history as the correction direction.
  • ERC adds negligible overhead because it works on residuals rather than exhaustive layer-wise feature correction.
Eq.(5)-(7), §3.3 p.5; §4.3 p.8

Crown insight

The right abstraction is not “cache more”; it is keep errors from entering the history loop.

HGC prevents stale reuse when history changes; ERC prevents contaminated residuals from steering later corrections.

§3.2-3.3 p.4-5; Table 2 p.8
Limits

What the paper does not solve yet

  • Manual threshold tuning: ARCache still relies on `delta`; authors name threshold-free adaptive caching as future work.
  • Model scope: evaluated on three representative ARDMs, not a claim for all non-autoregressive / non-diffusion video generators.
  • Quality-speed policy: high `delta` can reach 2.88x on FramePack-F1 but visibly lowers PSNR / LPIPS in ablation.
For production, delta should probably be selected per model, horizon length, and acceptable late-frame degradation.
§5 p.8; Table 2 p.8; §4.1 p.6
Engineer’s reading

How I would apply this idea

  • Instrument ARDM serving with per-segment history-token drift and late-frame fidelity curves; average latency alone hides the failure mode.
  • Treat cache refresh as a policy over history state change, not only timestep index or input norm.
  • Keep a clean reference trajectory / calibration anchor for early segments; avoid correcting long-horizon states using their own accumulated errors.
  • Expose `delta` as an explicit speed-quality control, then autotune against VBench-like task score plus late-frame consistency.

Appendix · Original highlight

Unlike SDMs, ARDMs generate each video segment conditioned on previously synthesized content; consequently, any approximation errors introduced by acceleration tend to propagate and accumulate over successive frames.

不同於 SDM，ARDM 每個片段都條件於已生成內容；因此加速帶來的任何近似誤差都會沿後續片段傳播並累積。§1 p.2

Our approach specifically monitors variations within the history tokens to trigger cache refreshes.

ARCache 不是看整體輸入變化，而是專門監控 history tokens 的變化來觸發 cache refresh。§3.2 p.4

Appendix · Original highlight

Rather than relying on the contaminated trajectory of the current segment, we leverage this inter-segment similarity and use the residual trajectory of the first segment as a clean reference to correct subsequent ones.

與其依賴當前片段已被污染的軌跡，ERC 利用跨片段相似性，以第一段 residual trajectory 作乾淨參考來校正後續片段。§3.3 p.5

Increasing delta from 0.10 to 0.30 significantly improves acceleration (from 1.52x to 2.88x speedup), but this comes at the expense of reduced visual quality.
