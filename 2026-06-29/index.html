DQO: 用 DPP 讓 RL post-training
不要把 LLM 壓成單一答案

Yilei Chen · Souradip Chakraborty · Lorenz Wolf · Yannis Paschalidis · Aldo Pacchiano
ICLR 2026 PosterarXiv:2509.04784v3PPO/GRPO-compatible
主角 paper: Post-training Large Language Models for Diverse High-Quality Responses
cover p.1; OpenReview BV3bLKaJ3Y
TL;DR

DQO 不是「把 temperature 調高」；它把多個 responses 的 semantic volume 放進 RL objective。

每個 prompt 取 k 個 responses，embedding 後用 kernel matrix determinant 衡量 group diversity；reward 控制向量長度，diversity 控制方向展開。

Abstract p.1; §3.1–3.2 p.4–5
Problem

RL post-training 的副作用：quality 上升，response modes 變窄

  • PPO / GRPO / RLHF 能提升 targeted task reward，但常讓輸出收斂到 narrow canonical responses。
  • 這會限制 reasoning alternatives、personalization、test-time search、robustness 與 exploration。
  • temperature / top-k 只是在 inference time 撒 token-level noise；常傷 coherence，也無法恢復 base model 沒學到的 modes。
  • training-time diversity 才是核心，但 lexical entropy / pairwise distance 都可能抓不到真正語義多樣性。
問題不是「答案不夠花」；是 post-training objective 本身把分布壓扁了。
§1 p.2 lines 65–97
Contribution

這篇 paper 的 4 個有效貢獻

  1. DPP-based semantic diversity: 用 response embeddings 的 determinant 作 group-level diversity。
  2. Quality-diversity RL objective: 可疊在 PPO/GRPO，上面加 alpha log det term。
  3. 幾何解釋: reward 是 norm，semantic embedding 是 direction；最大化 volume 同時要求高品質與分散。
  4. 可訓練化修正: det(L+I) bounded + leave-one-out estimator 降 variance。

Method · figure-first

DQO pipeline: 同一 prompt 取多個 response，再用 DPP diversity 更新 policy

  • policy model samples y1…yk
  • reward model 提供 quality signal
  • embedding kernel determinant 提供 semantic diversity signal
  • RL algorithm 用合成 objective 更新 policy
Fig.2 p.4; §3 p.4
Method · DPP metric

為什麼 determinant 比 pairwise distance 更難被騙

Div(y1:k) = det(Lphi(y1:k)),   L[i,j] = f(phi(yi), phi(yj))
  • embedding space 捕捉 semantic 而非 surface tokens
  • determinant 對 linear dependence 敏感
  • 如果 responses 擠在幾個 cluster，Gram determinant 會變小
  • pairwise distance 可能偏好「兩團很遠」的假多樣性
  • determinant 要求整組向量張成高體積
  • 這等於鼓勵多個互補語義方向同時存在
§3.1 p.4 lines 203–227
Mechanism crown

DQO 的核心不是 entropy，而是 volume under reward-weighted semantics。

高 reward 但語義重複，只會增加同方向長度；語義不同但低 reward，向量太短。能撐起 volume 的，是「高品質且方向分散」的一組 responses。

§3.2 p.5 lines 261–284
Objective

Quality-diversity objective：把 log-det 直接加進 RL

JDiv(pitheta) = E[sumi=1..k r(x,yi) + alpha log det(Lphi(y1:k)) - beta KL(pitheta || piref)]
  • alpha 控制 diversity term 的權重。
  • 最優 policy 可寫成 response group 的 determinant-proportional distribution。
  • 與 D-optimal design 類比：最大化 Gram determinant 等於選資訊量高、線性獨立的 points。
§3.2 p.4–5 lines 230–311; Appendix B p.18
Training stabilizers

直接 log det 很危險：paper 用兩個穩定器讓它可訓練

1. det(L + I)

  • 避免 det(L) 接近 0 時 log-det 變成巨大負值。
  • 可視為 regularization；值被 bound，alpha 更容易調。

2. leave-one-out estimator

  • 降低 k responses 梯度相加造成的 variance。
  • diversity term bounded in [0, log(1+k)]。
這是 paper 真正「工程可用」的部分：DPP idea 本身不難，穩定化才是關鍵。
§3.3 p.5–6 lines 314–362

Reviewer lens

審稿分裂很有資訊量：6 / 2 / 6 / 6

肯定

  • 問題重要：RL post-training diversity collapse。
  • DPP motivation 清楚，可接 PPO/GRPO。
  • 跨任務實驗方向完整。

質疑

  • reward model reliance 讓 reasoning/coding outcome reward 場景受限。
  • semantic diversity 起初過度依賴 LLM-as-a-judge。
  • 需要 embedding sensitivity、成本、larger models。
最尖銳的一刀是 reward hacking：這不是 appendix 小問題，而是方法適用邊界。
OpenReview reviewer JSON; Appendix G p.24
Limitations

這篇 paper 可以用，但不能無腦套到所有 RL reasoning

  • Outcome reward hacking: GSM8K 只用 outcome reward 時，模型會先答對、再加無關內容灌 diversity。
  • Embedding dependence: semantic diversity 由 frozen encoder 定義；encoder 不懂的差異，DQO 也看不見。
  • Scale gap: 實驗是 1B/1.5B 級，reviewer 明確要求 7B/13B。
  • Evaluation gap: human study 有補，但只有 18 prompts，仍偏小。
§5 p.10; App.F p.23; App.G p.24; reviewer JSON
Engineering interpretation

如果你要做 LLM 產品，DQO 最適合「需要多解、多風格、多候選」的 post-training。

例如 creative writing、summary variants、recommendation, brainstorming, best-of-n reasoning。若任務只能用 hard outcome reward，先處理 reward hacking，再談 diversity objective。
