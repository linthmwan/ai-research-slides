TimesFM: decoder-only foundation model for time-series forecasting

Abhimanyu Das, Weihao Kong, Rajat Sen, Yichen Zhou - Google Research
Repo: google-research/timesfmICML 2024 paperCaptured 2026-06-21

今天選題走週日 repo/report 路線：TimesFM 不是單純熱門套件，而是有 peer-reviewed paper、open checkpoints、2026 repo updates 的 forecasting foundation model。

Sources: GitHub Trending scrape 2026-06-21; README lines 6-19; arXiv:2310.10688v4 p.1
TL;DR

把 time-series patch 當作 token，用 decoder-only Transformer 做 zero-shot forecasting。

200M parameters + O(100B) real/synthetic timepoints，讓單一模型在 Monash / Darts / ETT 上接近或超過 per-dataset supervised baselines。

Problem

Forecasting 想要 foundation model，但 time series 比 text 更不規整

  • 傳統 ARIMA / ETS 或 supervised neural forecaster 多半要針對每個 dataset 訓練或調參。
  • Time series 沒有固定 vocabulary / grammar；context length、horizon、granularity 也都會變。
  • 公開 pretraining corpus 不像 text 那麼自然大量可得，還要覆蓋 hourly / daily / weekly / monthly 等週期。
  • 目標不是「更大的 GPT 拿來猜數列」，而是從 time-series-only corpus 學到可遷移 temporal patterns。
Introduction p.1 lines 24-46; Related Work p.2 lines 81-96
Core design

TimesFM 的兩個支點：資料多樣性 + decoder-only patch objective

  1. Large-scale corpus: Google Trends、Wiki Pageviews、public forecasting datasets、synthetic generators。
  2. Patch tokens: 把連續數值切成 patch，經 residual block 轉成 Transformer token。
  3. Causal next-patch training: 像 LLM 一樣平行訓練所有 prefix 的下一段預測。
  4. Longer output patch: h > p，減少 long horizon autoregressive steps。
Intro p.2 lines 53-60; Model Architecture p.3-5 lines 115-231
Architecture figure

Figure-first: decoder-only 不是口號，是訓練訊號的形狀

  • 每個 output token 只 attend 到過去 patch；因此不同 prefix length 的 forecasts 可在同一 batch 內訓練。
  • SA = causal self-attention；FFN = transformer feed-forward network。
Figure 1 p.4; §4 p.3-4 lines 153-197
Patch as token

數值 patch 進入 Transformer 之前，先被 residual block 嵌入

tj = InputResidualBlock(y~j * (1 - m~j)) + PEj
  • p = input_patch_len；第 j 個 patch 是 yp(j-1)+1 : pj。
  • mask m 讓 padding / missing prefix 不被注意力誤讀。
  • 200M base: 20 layers, model_dim 1280, 16 heads, p=32, output_patch_len=128。
§4 p.3-4 lines 153-197; Table 6 p.16 lines 1530-1553
Key deviation from LLMs

Forecasting 不該一格一格吐；TimesFM 讓 output patch 比 input patch 更長。

p=32, h=128 時，256-step horizon 只需 2 次 autoregressive generation；若 h=p=32，則要 8 次。

§4 p.3 lines 131-145; inference p.5 lines 227-231
Variable context

Random prefix masking 讓模型看過所有 context length

  • 每條 time series sample 一個 r in [0, p-1]，把第一個 patch 的前 r 點 mask 掉。
  • 若 max context=512, p=32, r=4，第一個 output 其實是在 28 個有效點後做 forecast。
  • 對所有 r 重複後，模型覆蓋 1 到 max context 的所有有效長度，而不必枚舉所有 window。
這是 TimesFM 比一般 encoder-decoder transfer 更適合 zero-shot variable-context 的機制之一。

Evaluation setup

三組 zero-shot benchmark，測的是跨 domain / granularity 泛化

  • Monash: 18 filtered datasets after excluding missing values; minutes to years, finance/demand/weather/traffic domains。
  • Darts: 8 univariate datasets with seasonalities and additive/multiplicative trends。
  • Informer / ETT: ETTh1, ETTh2, ETTm1, ETTm2; horizons 96 and 192; context length 512。
  • Baselines include statistical, supervised deep, PatchTST, and zero-shot llmtime。

Mechanism crown

TimesFM work 的真正原因：不是「Transformer 魔法」，而是 patch geometry 讓任意 horizon/context 可以被同一 objective 覆蓋。

  • output_patch_len 變長 -> 512-step forecast 的平均 MAE 隨 h=8..128 單調下降。
  • input_patch_len p=16/32 最佳；p=32 幾乎比 p=16 快兩倍，品質/速度折衷更合理。
  • synthetic data 補足稀有 granularity，尤其改善 15-min ETTm 與 Monash underrepresented periods。
§6.2 p.8-9 lines 466-499
Synthetic data

合成資料的角色：補真實 corpus 沒有的週期結構

  • Generator 包含 piece-wise linear trend、ARMA(p,q)、sine/cosine seasonal patterns。
  • 每條 synthetic series 長度 2048，隨機啟用/停用 components 並加權相加。
  • 作者指出 no-synthetic 模型在 Monash 與 15-min ETTm 掉分，但 hourly ETTh 幾乎不變。
  • 這暗示 synthetic data 補的是 granularity coverage，不只是 data volume。

Limitations

Paper 自承的未解題：probabilistic、covariates、interpretability

  • Probabilistic forecasting 可用 quantile / likelihood heads，但 paper 主體只做 point forecasting。
  • Covariates 沒有進 pretraining；作者只提出 residual regression 或 fine-tuning-time covariate inputs。
  • Deep foundation forecaster 的 interpretability 弱於 ARIMA/ETS；lag attribution 只能部分補。
  • Critical use cases 應 human-in-the-loop 或大量 testing/fine-tuning，作者也提醒可能 fail / hallucinate。
A.1 p.13 lines 671-699; Impact Statement p.9 lines 529-531
Engineering takeaways

如果把 TimesFM 放進產品，最先要想清楚三件事

  1. Granularity routing: hourly/daily/monthly context 長度不同，不能只套一個 max_context。
  2. Exogenous variables: 目前 repo 有 XReg covariate support，但 paper 本體沒有解決 universal covariate representation。
  3. Uncertainty UX: TimesFM 2.5 quantile head 很重要，因為 point forecast 在供應鏈/財務場景很容易給過度確定感。
對圖像/影片工程師的類比：這像把 temporal patch tokenization + autoregressive chunk decoding 搬到純數值時序。
